Collision check options

Use the following options to control the collision check.

Note

The tolerance values/limit values for the still permissible contact of the tool with the part/stock must be in reasonable proportion to the size of the tool used. OPEN MIND recommends defining these values no larger than half the tool diameter.

Check tool against model: Checks the tool against the model.

Check holder/shaft/core against model: Holder, shank and tool core are checked against the model.

Always check G0: The rapid traverse check is activated by default. If the Check tool against model function is deactivated, the check for all fast travel movements can also be optionally switched off. This can make it easier to approve an NC file in certain situations (for example, when machining non-modelled areas).

Check stock: Checks tool and holder against the stock.

Remove detached stock: Only available if Check stock is enabled. If this function is activated, a check is performed during the collision check and the simulation to see whether there is a stock area that has no contact with the defined milling area (model). If this is the case, the stock is automatically removed during the simulation and the collision check.

Consider core diameter for milling tools: The tool tip is checked for collisions with the stock, taking the core diameter into account.

Limits for part contact

Use G0 tip limit / G1 tip limit to set the limit value for the still permitted contact of the tool tip with the model in G0 and G1 movements. If a larger value is detected during the test, a message is generated.

Part allowance

Use part allowance: Activate if a part allowance should be defined for the collision check. The part allowance is then defined either using the settings in the job (if the Use job settings option has been activated) or it can be entered directly under Allowance. The tool shape is offset in all directions by the entered value. If -0.01 is entered, a ball mill D10 becomes a ball mill D9.98 during the collision check.

Use job settings: Enabled by default. The values defined in the job for allowance, allowance XY and allowance Z are used for the collision check.

Tip

A check is also performed to see whether a negative allowance is defined or the Chamfer height parameter is defined for the Chamfer Milling on 3D Model cycle. As a component violation is desired here, no collision message is output in these cases.

If the Use job settings option has not been activated, the value for the Allowance can be entered directly. The value that is entered is offset on the shank diameter. If -0.01 is entered, the diameter of the shank is reduced by 0.02 (shank radius -0.01).

Only tip: The allowance is only applied to the tool tip (to be activated if, for example, only a small area at the bottom of a cavity is to be machined with a negative allowance).

Whole tool: The stock allowance is applied to the entire tool (including thickened shank and holder) (to be activated if, for example, a complete steep area is to be machined from top to bottom with a negative allowance).

Stock contact

Allow thin shank contact: Activate to use a tool where the shank and cutting edge have been defined with the same diameter. The tool shank may touch the stock. No collision message is output. Use for imprecisely defined tool geometries.

derive from stock model resolution: Activate to use a tool where the shank and cutting edge have been defined with the same diameter. Situations in which there is a distance of 0 between the stock and shank are not regarded as a collision. The tolerance required for the collision check is automatically derived from the stock tolerance if the function is activated.

If the function is not activated, the value for the tolerance can be entered directly. The value that is entered is offset on the shank diameter. If -0.01 is entered, the diameter of the shank is reduced by 0.02 (shank radius -0.01).

Part contact

Allow thin shank contact: Activate to use a tool where the shank and cutting edge have been defined with the same diameter. The tool shank may touch the part. No collision message is output. Use for imprecisely defined tool geometries.

derive from job settings: Activate to use a tool where the shank and cutting edge have been defined with the same diameter. Situations in which there is a distance of 0 between the part and shank are not regarded as a collision. The tolerance required for the collision check is automatically derived from the calculation tolerance in the hyperMIL job if the function is activated.

If the function is not activated, the value for the tolerance can be entered directly. The value that is entered is offset on the shank diameter. If -0.01 is entered, the diameter of the shank is reduced by 0.02 (shank radius -0.01).

Tool safety distance to fixture

Clearance distance to be maintained between the tool and the fixture (exception: touch probe).

Use hyperMILL settings: The default value is defined in the hyperMILL dialog Joblist → Models → Fixtures → Tool safe distance.

Tool safety distance for MCS movements

Safety distance of the tool to be maintained for movements in the machine coordinate system (MCS, M91, ..).

Use VM configuration setting: The default value is defined in the VM configuration (Tool length tolerance → Upper tolerance).

Further collision check options

Resolution: Set the accuracy for the collision check.

Stop on first collision: During the collision check, the program stops at the first collision found.

Save As Default: The settings for the collision check are saved across machines and simulations (for the VIRTUAL Machining Center, the SIMULATION Center and the toolpath preview). Saving is user-specific. It is still possible to temporarily change settings by confirming with OK.

Note

OPEN MIND advises against saving deactivated collision-critical settings such as Check tool against model, Check holder/shaft/core against model or Check stock as defaults.

Please contact your OPEN MIND support team if you would like to save company-wide standards.
