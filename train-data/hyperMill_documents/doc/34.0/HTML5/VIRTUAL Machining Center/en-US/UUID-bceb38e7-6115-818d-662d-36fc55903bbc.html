5. Program run (simulation)

In the Program run (simulation) operating mode, simulate the VNC program file or NC program file. Display the control elements for regulating the speed and configuring the block display. Configure and switch on the collision check. Enable the material removal simulation.

Simulation requirements

The following requirements must be met to be able to perform a program run (simulation):

  • A component (Model) must be defined for milling and drilling operations, see hyperMILL → Job list → Model → Model.

  • A component (Model) and a 2D turning contour must be defined for turning operations, see hyperMILL → Job list → Model → Turning model.

If this is not the case, the VIRTUAL Machining Center cannot be started. A corresponding message is displayed.

Configuring the program run: Configure the view in the split/unsplit window of the block display. The following displays are possible for the upper/lower window: VNC program file, NC program file, coordinate values, status messages (warnings, error messages, collision areas), no display (empty).

Controlling visibility in the graphics area: Show or hide elements in the graphics area.

Simulate the VNC or NC program file: Set the start and end of the program run, approach a position with the machine and analyze, start and stop the simulation, move the machine axes one step at a time and regulate the speed for G0 and G1 movements.

Collision check and material removal: Control the collision check and activate the stock removal simulation.

Approve NC file: For an NC file to be approved, a collision check must be carried out. No errors should occur during this collision check.
