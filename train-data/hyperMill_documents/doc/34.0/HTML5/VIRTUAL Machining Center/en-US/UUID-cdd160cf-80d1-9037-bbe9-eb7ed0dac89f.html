5.4. Collision check and material removal

Change the search direction of the quick navigation for finding warnings, collisions or breakpoints:

Forward , Backward .

Move the machine to the next warning. The button turns yellow if a warning (for example, violation of the axis limit) has occurred.

Move the machine to the next warning or next error. The button turns red if an error or an error and a warning have occurred.

Move the machine to the next collision area. The button turns yellow after the collision check has been carried out if a contact has been detected.

Move the machine to the next collision area. The button turns red after the collision check has been carried out on detecting the following:

  • A collision or a component violation

  • A collision and a component violation

  • A collision and a contact

  • A component violation and a contact

  • A collision, a component violation, and a contact

.

Move the machine to the next breakpoint.

Start the collision check. Analyze the loaded program for possible collisions. If a collision or part violation is detected, the collision area is highlighted (red) in the program file. The NC file is not approved.

Note

A collision check can be performed in the Project administration and Program run (simulation) operating modes. In the Project administration operating mode, all loaded main programs (NC list) are checked in chronological order, while in the Program run (simulation) operating mode, only the loaded main program is checked. After starting the VIRTUAL Machining Center, the first main program is loaded automatically.

Collision check options. Specify which components are checked during the collision check.

Simulate the removal of material in the program run.

A collision check must be carried out before a decision can be made about whether or not to approve an NC file.

Collisions occurred during the simulation. The NC file cannot be approved. For an NC file to be approved, a simulation must have been carried out without errors while the collision check is enabled.

Contacts or violations of the machine limits occurred during the simulation. To approve the file on the Contacts / Warnings tab, use the shortcut menu of the message (Allow release of programs with contacts). The NC file can now be approved.

No collisions occurred during the simulation. The NC file can be approved.

Click the button to save the NC file in the defined folder.

The NC file has been approved and is available in the defined folder.

Note

A simulation only takes place up to the safety position of the machine. A tool change is not simulated.
