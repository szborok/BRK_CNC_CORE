5.5. Approving the NC file

By default, an NC file is not available in any operating mode. For an NC file to be approved, a collision check must be carried out. This is symbolised by a corresponding approval button next to the control panel for the program run.

Click the button to carry out a collision check. No errors should occur during this collision check. Possible errors are: a collision or a part violation, a 180° rotation or a rotary axis rotation greater than 7.5°.

After the collision check has been carried out, a message with status information is output. The approval button turns green if there are no collisions or part violations.

The message includes the settings that were defined in the Collision check options dialog. When text is displayed in red and crossed out, this means that the respective element has not been checked.

To ensure that no collisions are present, all available elements must be checked.

OK: Close the dialog.

Edit: Open the generated NC file.

Open folder: Open the output directory in which the generated NC file was saved.

If a collision has been detected, the approval button turns red. If there is a warning, it turns yellow. Click the approval button to open the VM Release Manager dialog with further information:

The NC file cannot be approved for the following reasons: There are collisions. Click Show details to display the collisions detected and the line numbers in the display area.

The NC file cannot be approved for the following reasons: There are warnings. Click Show details to display the warnings, such as violations of the limit value, in the display area along with the line numbers.

Click Allow release to allow an approval despite warnings. The following message appears: The NC file can be approved and the approval button turns green.

The NC file has been approved and is available in the defined folder.

Definitions

The following applies to the terms used in the collision check message:

Collision: Collision between part, stock, fixture and machine or the non-cutting part of the tool (the tolerance values are defined in the dialog Collision check options → Limits for part contact). An NC program with collisions cannot be approved.

Gouge: Violation of the part with the cutting area of the tool. An NC program with part violations cannot be approved.

Contact: Permitted violation of the part with the tool tip (the tolerance values are defined in the dialog Collision check options → Limits for part contact). An NC program with contacts can be approved.

Important

If the cutting edge length is 0, the length of the tool tip corresponds to the cutting tool part. If the cutting edge length is not equal to 0, the length of the tool tip without cutting edge corresponds to the non-cutting tool part.
