4.1. Manage project

Manage main programs, tool change jobs and operations in a project.

Main program: Program created by the postprocessor. In Single mode, based on an individual hyperMILL job. When transferring a job list, compound or linking job with several jobs, a main program is created for each job in the job list (linking job/compound job). In Package mode, the complete job list (linking job/compound job) is converted to a main program.

Tool change job: position for tool change. Part of each main program.

To simulate a loaded, active main program. All other main programs are greyed out.

Selected entry in the NC list.

This job only: entry in the shortcut menu of main program, tool change job or operation. Limit the program run to the selected job and the display of the toolpath. All unselected jobs in the list are greyed out. The start and end points in the program run are adapted synchronously for the list display. If you move the start and end points manually, the respective jobs will also be displayed ‘normally’ in the project administration while all other jobs are greyed out.

Run to start of job: entry in the shortcut menu of main program, tool change job or operation. Move the simulation to the beginning of the selected job.

Load this program: entry in the shortcut menu of the main program. Load the main program for the program run. The main program is then active. Only one main program may be active at once.

Show postprocessor log: entry in the shortcut menu of main program, tool change job or operation. Display the postprocessor messages for this job in a text editor. The information can be saved as a text file (.txt).

Simulate complete NC list: Starting with the main program selected in the list (active), all main programs are simulated in sequence.

Stop at the end of a program: The simulation stops after every main program.
