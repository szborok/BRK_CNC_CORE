Information in the display area

The following example shows information from the display area that is available during the simulation or after the collision check has been performed.

  • The upper window of the display area shows the status messages , separated into the following areas: Collisions, Gouges, Contacts, Errors and Warnings. The program lines, in which a collision, part violation or contact was detected, are listed on the Collisions, Gouges and Contacts dialog tabs. Status messages, which refer to the simulation of the VNC or NC program file, are output on the Errors and Warnings dialog tabs.

  • The lower window of the display area shows the block display .

Status messages

A status message is output, among other things:

  • if a component is in collision,

  • an axis is outside of the permissible limits,

  • an NC block cannot be interpolated reliably and can therefore not be checked for collisions (for example, in 180° rotations),

  • a transformation is to be carried out that is not possible with the machine kinematics used.

Warning

As only the axis positions at the start and end point of a block are checked when a VNC program file is read, a collision check must be performed in order to also display workspace violations that arise due to the fact that an axis is located between the start and end point of a block outside of the permissible workspace.

Collision between Shaft, Tip and Part (*).

Note

(*) The names of the components listed in the status messages depend on the names used in the machine model.

The following functions are available via the shortcut menu of the status messages (right mouse button):

Save Save information about the program lines with collisions to a text file.

Move to this line Position the simulation at the beginning of the collision area. In the block display, the respective blocks are shown in red, as is the component that is in collision in the program run.

Show details Display more detailed information about the components that are in collision.

Block display

Highlighted in yellow: The machine axes are out of limit: Z. Position the mouse pointer over the highlighting to display the warning.

Highlighted in red: A machine component is in collision with the part or stock.

The following functions are available via the shortcut menu of the block display (right mouse button):

Run to this position: Move the tool to the selected position.

Run to line: Move the tool to the position that has been entered.

Set start of simulation / Set end of simulation: Set the selected position as the start point or end point for the simulation.

This job only: Limit the program run and the toolpath display to the selected job. All unselected jobs in the list are greyed out. The start and end points in the program run are adapted synchronously for the list display. If you move the start and end points manually, the respective jobs will also be displayed ‘normally’ in the project administration while all other jobs are greyed out.

Run to start of job / Run to start of previous job: Move the simulation to the beginning of the selected or previous job.

Run to next tool change job / Run to previous tool change job: Move the simulation to the beginning of the next or previous tool change job.

Search: Search for coordinates or control commands in the VNC or NC program file by entering a search string. The search position that is found is highlighted in the program file. Find next Find the next block that contains the search string. Erase search marker: Delete a search marker in the VNC or NC program file.

Save current stock: Save the current stock in *.x format.

Note

Text shown in red in the block display: This is an internal error. This may be a message from the postprocessor, for example. Please contact OPEN MIND Support.

Switch rest material display on / Switch rest material display off: Switch the rest material display on or off.
