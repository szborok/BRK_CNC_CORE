4. Project administration

In the Project administration operating mode, display the main programs (VNC files), manage stock, move the setup, optimally fit the part (‘best-fit’), define collision check options and perform a collision check.

Manage project: List of all main programs, tool change jobs and operations.

The main program has not yet been checked for collisions. For approval purposes, a collision check must be carried out.

Following a successful collision check, the main program is marked as follows:

Collisions occurred.

Contacts occurred. Before the NC file is approved, the approval (of programs with contacts) must be allowed interactively.

Collisions did not occur. The file can be approved.

Collisions did not occur. The file has been approved and saved in the defined folder.

Manage stock: The used stock, part and clamp are displayed in a list.

Move setup: Move the setup (part, stock, clamp) in relation to a reference on the machine.

Best Fit: Align the part optimally on the machine using the information from the OPEN MIND probing protocol.

Collision check: Start the collision check and analyze all programs (NC list) for possible collisions.

Collision check options. Define the elements to be checked during the collision check.
