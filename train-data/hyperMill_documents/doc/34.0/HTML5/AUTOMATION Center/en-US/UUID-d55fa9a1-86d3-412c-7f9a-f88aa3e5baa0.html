2.2. Configuration

Process application

Runtime app folder: Select the folder in which scripts provided by OPEN MIND (Runtime or Customer Solution Reference apps) are stored to display them in the Customer applications branch. This selection overwrites the folder that has been selected in the default path.

Editor: Choose a text editor to edit text, for example, in a VBScript. If no selection is made, the Notepad++notepad++.exe software is searched for first. Otherwise, the Windows Editor is used.

Script base folder: Folder in which to store the scripts (in subfolders if necessary). In , you can select from the history.

Graphics editor: Select an image editing program.

Format: Control whether completed, licensed customer solutions should be displayed in the format *.xml or *.hms. This setting can only be specified with a Developer license. No change may be made with a BASIC license.

Functionality: Filter the scope of the functionality that is displayed in accordance with the selected license.

Display command names in Execute mode: Optionally display the function names in Execute mode – useful for test scenarios.

Display hidden parameters: Also display the parameters from the parameter definition for which the lock (in the first column of the table) is closed.

Ask reset runtime parameters: While a script is executed, settings are made that the script remembers, such as the selection of a particular machine. To work with the original settings of the script again, select this option. When the script is called again, a prompt appears asking whether the settings should be reset.

Ask closing AUTOMATION Center: Optionally confirm the closure of the AUTOMATION Center.

Update runtime scripts: A decision can be made whether to update existing local runtime scripts from the server.

Run inside AUTOMATION Center (no execute mode): For analysis purposes, the AUTOMATION Center is run with its dialogs to process a script.

Display text panel in execute mode: When executing a script, add the text field with the optionally stored instruction to the dialog of the script step.

Ignore stop points: Ignore the stop points set for developing.

Sample: Load milling toolkit, turning toolkit, and/or sample scripts.

Note

Always load the CAD model into the document before executing a script.

The Undo and Redo functions are not supported. Useful background information: CAD entities such as the geometry that specifies the fixture are converted into CAM entities. This removes them from the CAD entity management. These converted entities can only be deleted, for example, as a whole.
