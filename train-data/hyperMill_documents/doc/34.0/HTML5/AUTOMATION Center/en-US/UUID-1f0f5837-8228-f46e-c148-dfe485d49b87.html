Exact collision check (SIMULATION Center)

Perform a collision check.

Template commands → Basic Programming → Exact collision check (SIMULATION Center)

Basic

Template commands → CAM → Programming → Exact collision check (SIMULATION Center)

Advanced

Perform a collision check without displaying a simulation. The collision check is performed according to the options saved for the collision check. Collisions found are classified. It is possible to configure which types of collisions are to be reported/are reported. In case of collision, start either the VIRTUAL Machining Center or the SIMULATION Center. If a postprocessor is selected where the Virtual Machine option is not set, the SIMULATION Center will be started. If the option is set, the VIRTUAL Machining Center is started.

1. Definitions

The following applies to the terms used in the collision check message:

Collision: Collision between part, stock, fixture and machine or the non-cutting part of the tool (the tolerance values are defined in the dialog Collision check options → Limits for part contact). An NC program with collisions cannot be approved.

Gouge: Violation of the part with the cutting area of the tool. An NC program with part violations cannot be approved.

Contact: Permitted violation of the part with the tool tip (the tolerance values are defined in the dialog Collision check options → Limits for part contact). An NC program with contacts can be approved.

Important

If the cutting edge length is 0, the length of the tool tip corresponds to the cutting tool part. If the cutting edge length is not equal to 0, the length of the tool tip without cutting edge corresponds to the non-cutting tool part.

Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Joblist: Select a joblist for using the function.

Job range: Perform the calculation using the jobs defined in the job range of the current or last job list.

Define the job range in advance using either Define job range or Multi job selection, or define it directly. A selection made using Job range overwrites another selection, such as a job list.

  • Define an area that also contains all existing jobs within it: The following syntax is used for the start job: SID:, for example, SID:2. The following syntax is used for the end job: EID:, for example, EID:4. Insert a hyphen in between. This results, for example, in SID:2-EID:4.

  • List the jobs: The following syntax is used for the jobs: JID:. This results, for example, in JID:1; JID:4;. Separate the individual jobs with a semicolon.

CHECK: gouge: Report whether collisions have occurred that are classified as a component violation. In a component violation, the cutting part of the (milling) tool causes damage to the workpiece, for example, in the case of an infeed via the part. This renders the workpiece unusable.

CHECK: touch: Report whether collisions have occurred that are classified as a component contact. The tool tip (cutting area, non-cutting area of the tool, core) touches the workpiece. Note: This collision check is only performed if the Limits for component contact are set in the collision check options.

CHECK: collision: Report whether other collisions have occurred, that is, collisions that cannot be classified as a component contact or component violation.

CHECK: limits: Report whether machining can be carried out within the axis limits, that is, whether production is possible in the selected clamping position.

Start VM in case of danger: If at least one of the four checked collisions occurs, the VIRTUAL Machining Center or SIMULATION Center software is started and the data is loaded. Investigate the situation interactively.
