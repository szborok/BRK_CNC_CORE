Load generic CPF

Change values in the filter definition of a CPF.

Template commands → CAM → Feature → Load generic CPF

Advanced

Load a CPF defined as a pattern (abbreviation for Customized Process Feature) with a layer name adapted to the model situation. For this purpose, layer names of the exported CPF file are changed before the file is imported and applied. This makes the CPF definition generally usable, that is, generic.

Run the Select feature folder function beforehand. Also see Load index CPF.

Example 38. Match a pattern CPF to a clamping situation with four setups on a machine table that are organized on four layers.

  1. All model data for the four setups are organized on four layers named Model1, Model2, Model3 and Model4.

  2. If not already available, define a CPF that serves as a pattern. In the filter definition for a layer filter in Filter definition → CAD attribute → Use layer filter → Layer filter → Layer, enter the layer name Model. Export the CPF. To do this, open the context menu after the definition by holding down the CTRL key and select the Export features function. Save to a subfolder OwnCPFs in the Default folder for the AUTOMATION Center in ...\FEATURE as Example.fst. Create the subfolder if it does not already exist.

  3. Insert the Select feature folder function into the script and execute.

    Input: Feature folder: OwnCPFs

  4. Insert the Load generic CPF function into the script, fill and execute.

    Input: Generic feature: Example| Feature name: Feature1 | Replace 1: Model | New value: Model1

  5. Repeat this step three times, replacing the 1 with 2, 3 or 4.

As results, the CPF features Feature1, Feature2, Feature3 and Feature4 are created in the feature browser and the data of the layers Model1, Model2, Model3 and Model4 are filtered out for the respective feature.



Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Generic feature: If there are generic features in the project, a drop-down menu for selecting a generic feature is available. Execute the Select feature folder command in the script. There must be exported features in this folder.

Feature name: Enter the feature name for the new feature that is derived from the CPF to be loaded.

Feature class: Enter a feature class for the new feature.

Replace 1 to Replace 30: Enter existing layer names (Filter definition → CAD attribute → Use layer → Layer filter → Layer) and, if necessary, the name of the filter (Filter definition → General → Name) in the CPF to be loaded if they should be replaced by a new value.

New value 1 toNew value 30: Enter the new values for layer names and filter names.

Tip

To view the changed filter, highlight the newly created CPF feature and select the Edit function on the context menu and Not the Edit CPF definition function.
