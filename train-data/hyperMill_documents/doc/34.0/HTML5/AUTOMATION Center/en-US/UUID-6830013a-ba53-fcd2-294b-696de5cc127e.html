Multi job selection

Select one job, or manually select several jobs, (of a selected compound job) from the (last) joblist.

Template commands → AC general → User interface → Multi job selection

Advanced

Select one job, or manually select several jobs, (of a selected compound job) from the (last) joblist. A stop is forced if more than one job exists. Otherwise, the only existing job is enabled without a stop. Also see Is job / joblist selected.

Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Mode: Select from the drop-down menu whether all jobs (All on) or no job (All off) or the saved selection of jobs (Saved selection) are preselected in the display of existing jobs.

Compound job: Optionally select a compound job. Enter the name.

: Display the existing jobs (of a selected compound job) of the (last) joblist for selection. Select one or more jobs.

Stop for input

Execution of the script is halted so that information can be entered.

Job range: The selected jobs are output. The following syntax is used for the jobs: JID:. This results, for example, in JID:1; JID:4;. The individual jobs are separated by semicolons. Transfer the selection to another command with the help of the script variable, for example.
