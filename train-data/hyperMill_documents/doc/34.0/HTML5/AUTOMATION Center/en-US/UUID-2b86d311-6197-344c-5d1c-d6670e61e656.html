Define server mode

Generate additional information about the state during the calculation on the server. This information is output to the monitor.

Template commands → AC general → Client / server → Server → Define server mode

Advanced

Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Watch point name: Enter a text that is to be passed to the monitor as information on the state of the calculation. This could include information such as ‘Feature Recognition started’ or it may be a message that points out script steps that require considerable processing power.

State: Enter a text describing the state of the server.

Icon: Select an icon that matches the state (Ok, Waiting, Finished, Error, Collision).

Error folder: Specify an error folder as a default setting.

Add user name to path

Create a subfolder with the user name.

User name: Overwrite the stored user name.

Monitor folder: Select folder for monitor file. Different folders are required, including one folder for information that should go to the server and another folder for information that should go from the server to the client.

Monitor filename: A file in which the current states of the calculation are entered to make it possible to monitor the server actions on the client. At the start of a return transfer, for example, information indicating that the calculation on the server has finished successfully.
