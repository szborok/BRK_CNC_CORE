Call subprogram script

Continue with another script. Call a script as a subroutine.

Template commands → Basic Programming → Call subprogram script

Basic

Template commands → AC general → External automation → Call subprogram script

Advanced

After the script has been called, the process continues in the parent script. In general, a check is made at the end of each script run to see whether it was called as a subprogram from a parent script. Where necessary, the process returns to the position of the calling script.

Up to 20 values can be passed to the child process. The parameter names of these values are $FUNCTION_PARAM1# to $FUNCTION_PARAM20#.

Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Script name: Enter a script name as it appears in the AUTOMATION Center in the drop-down menu for script selection.

Subscript: Select a subscript to be executed if a subscript is defined within the selected script.

For up to 20 parameters, enter the descriptions of the parameters in the dialog that (for the first time in a second step) are added to the first column of the table as names and can be assigned values:

This is useful if a subscript is not to be used with global parameters of the master script, but repeatedly with other values in the script run, for example, in a loop.

Parameter 1 description to Parameter 20 description: Enter short descriptions for the parameters.
