Optimize tools (VT)

Optimize tools (VT).

Template commands → CAM → Tool → Optimize tools (VT)

Advanced

VT stands for ‘Virtual Tool’. A suitable tool is searched for from the tool database with the help of the *.vtx file extended for the AUTOMATION Center.

Example 45. Overview of the arrangement of functions for optimizing the tool projection length.

Optimization of the tool projection length to avoid tool holder collisions, as in here for example, for a hole inside a pocket.

In preparation, a user database project with a macro database and a Virtual Tool must be entered in hyperMILL → Setup → Settings in Database.

  1. Execute Prepare VT XML for AC. The *.vtx from the Virtual Tool is supplemented with data structures and so prepared for processing in the AUTOMATION Center.

  2. Then, for example, run a hole feature recognition process and assign a macro. Drilling jobs are created.

  3. Execute Prepare tools and jobs for minimal tool length. In hyperMILL® in the job, in Settings in Calculate length for a tool, the option Reduce is activated. The specified imaginary tool length is set for all jobs or only for the jobs with a collision warning.

  4. Recalculate jobs. In the newly calculated jobs, the imaginary tools are assigned the minimum projection length that has now been reduced again.

  5. Execute Prepare tools and jobs for minimal tool length. In hyperMILL® in the job, in Settings in Calculate length for a tool, deactivate the Reduce option.

  6. Execute Calculated minimal tool length to job CGF. It goes through the jobs. The minimum projection length is transferred to the CFG parameter of the respective job.

  7. Execute Optimize tools (VT). A suitable tool is searched for from the tool database with the help of the *.vtx extended for the AUTOMATION Center.

  8. Recalculate jobs.

  9. Delete data that is no longer required.
