Definition mode

Switch to definition mode in the main dialog using the button. Prepare a script. All functions which can be inserted into a script are listed in a tree structure on the right side of the dialog, and are known as template functions. Already pre-configured scripts also appear there.

The scope of available template commands and pre-configured scripts depends on the purchased license.

Figure 2. Schematic representation of the main dialog with definition mode switched on


All available template commands and pre-configured scripts are shown.

Collapse all

Collapse the structure tree on the context menu.

Expand all

Expand the structure tree on the context menu.

Add to favorite selection

Mark the template command as a favorite on the context menu.

Remove from favorite selection

Remove the template command from the favorite selection on the context menu.

Optionally display only the template commands of the favorite selection .

Insert functions and components into the script:

Create a new component.

A component is necessary to insert functions into the script.

Create a subcomponent.

Transfer the template command or component selected on the right side of the dialog into the script on the left side.

To do so, select the function or pre-configured component in the right column and insert it in the left column using the button. This is also possible by double-clicking with the left mouse button or dragging and dropping.

Edit the script:

Change to subscript

A component can be converted into a subscript. To do this, click the desired component on the left side and right-click to open the context menu. Select the Change to subscript function. If necessary, change the name for the subscript in the editable line below. To do this, click the line.

Copy functions within the script. Copying is only available for functions, components cannot be copied.

Delete the selected function or component without notification.

Switch off a selected script step.

Undo or redo the last change in the script.

Change the sequence of functions and components in the script:

Move the component or function one step up. Move a function from one component into another.

Move the component or function one step down. Move a function from one component into another.

Move a component to the start of the script or move functions up within a component. Use step movement to move functions from one component to another.

Move a component to the end of the script or move functions within a component to the end. Use step movement to move functions from one component to another.

Export or import a component or subcomponent of a script. Frequently used script steps can be easily reused. A sequence of script steps can be moved to another script. Only the applicable parts of a script can be sent to support. A component is saved including all user localizations and parameters:

Export component

Click the required component on the left side of the dialog and right-click to open the context menu. Select the Export component command. Export the component in *.ACcomp format. Enter a file name. The ...\COMPONENTS file folder is preset as the default folder.

Import component

Click the required command or component on the left side of the dialog and right-click to open the context menu. Select the Import component command. Import a component in *.ACcomp format. Select a file. The ...\COMPONENTS file folder is preset as the default folder.

If you have been working on a project for a long time, there may be many parameters that are no longer used. Often because the corresponding commands have been deleted from the script. These parameters that are no longer used can be removed with a cleanup. The loading time of the script is accelerated.

Clean up parameters

Click the required component on the left side of the dialog and right-click to open the context menu. Select the Clean up parameters command.

Important

Check the script after the cleanup. It is recommended to create a backup copy and export the script beforehand.
