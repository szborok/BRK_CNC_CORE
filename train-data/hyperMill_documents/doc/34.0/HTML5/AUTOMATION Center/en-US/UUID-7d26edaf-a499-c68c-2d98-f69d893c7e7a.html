Complex loop

At the beginning, the type of loop is defined using a condition.

Template commands → AC general → Helper functions → Complex loop

Advanced

Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Type: Select a mode of behavior for the component.

Container

A component container is used for structuring. The functions contained are processed in their specific sequence.

Linked functions

Functions with the blue icon can be linked; other functions, such as those with loops, cannot be linked.

Layer loop

Contained functions are executed for the filtered layer, for example, for the respective current layer $CURRENT_LAYER#.

Filter layer: Filter instruction, for example, enter a part of layer names.

Counted loops

Contained functions are executed x times. The current number of passes is in $CURRENT_LOOP_INDEX#.

Number of repeats: Enter a number.

Counted loops 2

Contained functions are executed Y times, for example, for setting up a double loop, where a loop is passed through X times. And inside, a second loop is passed through Y times for each pass. The current number of passes is in $CURRENT_LOOP_INDEX2#.

Number of repeats 2: Enter a number.

Joblist loop

Contained functions are executed for joblists that meet the condition.

Include spindles: Include main spindle and counter spindle jobs for turning.

Job loop and Job Loop 2

Contained functions are executed for the filtered job, for example, for the respective current job $CURRENT_JOB#.

Joblist filter: Filter instruction, for example, enter a part of a joblist name.

Sub job list filter: Filter instruction, for example, enter a part of a sub-joblist name.

Filter jobs: Filter instruction, for example, enter a part of a job name.

Only compound jobs: Optionally execute only for compound jobs.

Only active jobs: Optionally execute only for active jobs. A job is active when the Create NC file option of the General dialog page is enabled in a job.

Tool filter: Run through for milling tools, turning tools or all tools.

Compound job: Enter compound job.

Include spindles: Include main spindle and counter spindle jobs for turning.

Job tool loop

Runs through all jobs and makes one pass for each existing tool.

Joblist filter: Filter instruction, for example, enter a part of a joblist name.

Sub job list filter: Filter instruction, for example, enter a part of a sub-joblist name.

Only active jobs: Optionally execute only for active jobs. A job is active when the Create NC file option of the General dialog page is enabled in a job.

Order: Run through the order of the NC number from the tool definition in ascending or descending order of appearance in the job list.

Include spindles: Yes / No

Tool loop

Contained functions are executed for the filtered tool, for example, for the respective current tool $CURRENT_TOOL#.

Order: Run through the order of the NC number from the tool definition in ascending or descending order of appearance in the job list.

Joblist filter: Filter instruction, for example, enter a part of a joblist name.

Tool filter: Run through for milling tools, turning tools or all tools.

Used tools only: Only run through for used tools.

Order: Processing takes place in the selected order of appearance or in the selected NC number order.

Feature loop

Contained functions are executed for the filtered feature, for example, for the respective current feature $CURRENT_FEATURE_LOOP_INDEX#.

Feature filter: Only run through for drill features or pocket features.

Frame loop

Contained functions are executed for the filtered hyperMILL® frames, origins, or both sub-types, such as for the current frame $CURRENT_FRAME#.

Sub type: Execute for frames, origins, or both sub-types.

Filter frame: Filter instruction, for example, enter a part of a frame name.

File loop

For running through all files with a selected file extension Extension in a selected Folder.

File filter: Filter instruction, for example, enter a part of file names.

Entity loop

Go through all entities that are filtered out using Entities color and Entities layer.

Filter type: Process a selected entity type (for example, faces, solids, curves, point) or all these entity types.

Bookmark loop

Contained functions are executed for the states stored by the filtered bookmarks, for example, for the respective current bookmark §CURRENT_BOOKMARK_LOOP_INDEX#.

Bookmark filter: Filter instruction, for example, enter a part of a bookmark name.

Workplane loop

Contained functions are executed for the filtered workplanes, for example, for the respective current workplane §CURRENT_WORKPLANE_LOOP_INDEX#.

Filter workplane: Filter instruction, for example, enter a part of the name of a workplane.

Text loop

Write all Excel worksheets to a text file. This text file can be used to create a combo box or a loop. Set up a loop with theRead Excel sheets into file command.

Use Folder and Filename to specify the text file.

Conditional container

A stop occurs before the component container is processed. The user must decide whether a condition applies. If the user chooses ‘Yes’, the functions within the container are executed. If the user selects ‘No’, the container is skipped (in contrast to conditions in other types of components, where there is no compulsory stop).

Question: Enter a question.

Loop condition: Check whether a script variable is <, >, = or ≠ to a value.

Type (alternative): Select alternative loop type.

Only active jobs: Optionally execute only for active jobs. A job is active when the Create NC file option of the General dialog page is enabled in a job.

Condition

Check whether a script variable is <, >, = or ≠ to a value.

Error level: Specify which information should be displayed as a message and at what time.

  • Off: No messages are output.

  • Basic: Messages are only output for serious errors.

  • Input: Messages are only output if the starting conditions necessary for the script step do not exist or are not correct.

  • Input / output: Messages are only output if the starting conditions necessary for the script step do not exist or are not correct or no result or no correct result can be generated.

  • Detailed: Debugging information is also output for testing purposes during software development.
