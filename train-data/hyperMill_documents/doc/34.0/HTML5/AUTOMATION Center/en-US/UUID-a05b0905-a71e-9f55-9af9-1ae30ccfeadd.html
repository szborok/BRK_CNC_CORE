Is job / joblist selected

Check if job or joblist is selected.

Template commands → AC general → User interface → Is job / joblist selected

Advanced

For example, make a selection with the functions Multi compound job selection, Multi joblist selection or Multi job selection and use Is job / joblist selected to check within a job loop whether the desired job or the desired joblist is selected.

Example 4. Provide properties of a job from the hyperMILL® job for the AUTOMATION Center

  1. Multi job selection

    The jobs from the last joblist in the job browser are offered for selection. Select a job:

    Input: : T6 3D Profil finishing

  2. Component

    Input: Type: Job loop

    Results: Each job in the job browser is checked to see whether it has been selected. If not, the check returns False=False and the loop is repeated until the check returns True=False:

    1. Is job / joblist selected

      Input: Job / joblist UUID: §CURRENT_JOB#

      Results: Result: True

    2. Continue loop

      Input: Condition: §SCRIPT_VALUE_#...=False

      Result: Condition: True=False

    3. Read job

      The selected properties are provided for the AUTOMATION Center.

Job / joblist UUID: In a job loop, check all existing UUIDs to see if they match a selected UUID of the above functions. The UUID checked in each case is displayed.

Result

The result of the calculation in the function. Transfer to other functions with the help of the script variable.
