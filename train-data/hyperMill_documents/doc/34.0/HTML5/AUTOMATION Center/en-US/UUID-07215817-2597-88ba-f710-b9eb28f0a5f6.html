Mark not complete cylinder hole feature faces

Mark not complete cylinder hole feature faces.

Template commands → CAM → Feature → Mark not complete cylinder hole feature faces

Advanced

Mark not complete cylinder hole feature faces. Incomplete features are features in which the volume is less than the volume that would result if the feature were completely enclosed by cylindrical faces along its entire length. This can be useful for checking whether a hole should be made before a pocket instead of after a pocket because it is not surrounded along its entire length by material with a wrap angle of more than 180°.

Example 42. Concept for dealing with incomplete features

  1. In Hole feature recognition settings, enable the option Tag used faces.

  2. Hole feature recognition.

  3. Execute the Mark not complete cylinder hole feature faces command.

    Cylinder faces are identified as part of a hole, and the Tag "OM_HOLES=omHole (Feature Mapping)" is assigned. The UUID of the relevant hole feature is assigned to these cylinder faces. The "FeatureUUID" is set and receives the UUID of the hole feature as a value.

    Input: Feature name add: NOT COMPLETE | Feature class add: NOT COMPLETE

    The incomplete feature is determined. Its feature name is supplemented with the addition "NOT COMPLETE". The feature class "NOT COMPLETE" is set.

  4. Filter out features based on the feature name component that were added using Feature name add in Define feature / macro filter in order to process them differently in the later script run.
