Load index CPF

Load CPF extended by index.

Template commands → CAM → Feature → Load index CPF

Advanced

Expand the filter names in the selected CPF (abbreviation for Customized Process Feature) with the currently set index when loading to be able to use the CPF several times in the same project.

Execute the Select feature folder command beforehand Determine the index using the Set current index command or the current index of a feature loop.

Also see Load generic CPF.

Example 40. Match a pattern CPF to a clamping situation with four setups on a machine table that are organized on four layers.

  1. All model data for the four setups are organized on four layers named Model1, Model2, Model3 and Model4.

  2. Define a CPF that serves as a pattern if one does not yet exist. In the filter definition for a layer filter in Filter definition → CAD attribute → Use layer → Layer filter → Layer, enter the layer name Model. Export the CPF. To do this, after defining the CPF, call the context menu while holding downCTRL and select the command Export feature. Save in a subfolder titled OwnCPFs in the Default folder for the AUTOMATION Center in ...\FEATURE as Example.fst. Create the subfolder if it does not already exist.

  3. Insert the Select feature folder function into the script and execute.

    Input: Feature folder: OwnCPFs

  4. Insert the Set current index function into the script and execute.

    Input: Index: 1

  5. Insert the Load CPF with index function into the script, fill and execute.

    Input: Generic feature: Example| Feature name: Feature$INDEX_#UUID | Replace 1: Model

  6. Repeat steps 4. and 5. three times, replacing the 1 with 2, 3 or 4.

As results, the CPF features Feature1, Feature2, Feature3 and Feature4 are created in the feature browser and the data of the layers Model1, Model2, Model3 and Model4 are filtered out for the respective feature.



Command name

Enter a brief explanation of the purpose of the function. This explanation is used as the name in the script.

The character | is not allowed to be used. A * is replaced by the original command name.

Generic feature: If there are generic features in the project, a drop-down menu for selecting a generic feature is available. Execute the Select feature folder command in the script. There must be exported features in this folder.

Feature name: Enter a new name for the CPF to be loaded.

Feature class: Enter a new feature class for the CPF to be loaded.

Replace 1 to Replace 20: Enter existing layer names in the CPF to be loaded (Filter definition → CAD attribute → Use layer filter → Layer filter → Layer) if they are to be supplemented by the index.
