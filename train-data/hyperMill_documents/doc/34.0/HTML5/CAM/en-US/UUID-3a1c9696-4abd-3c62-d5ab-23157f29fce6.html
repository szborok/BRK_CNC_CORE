NC event

Use NC events to clearly structure the program flow and influence programming by entering commands, independent of job lists and jobs.

To create an NC event:

  1. Use the New â†’ NC event function on the browser's shortcut menu.

  2. Define the Event type and enter the respective command under Event command.

NC events can be freely moved in the job browser. To do this, select the NC event with the left mouse button and move it to the required position while holding down the mouse button. The content of the NC Event always remains intact. Control-specific functions and commands are forwarded to the postprocessor directly by the NC Event.

Warning

NC events cannot be simulated or checked for possible collisions. Postprocessing is only possible in Package mode.

Free NC text is now also supported by VIRTUAL Machining. Only input in VNC format is permitted. Otherwise, an error message is generated in the VIRTUAL Machining Center. It is then no longer possible to release the NC program.

VIRTUAL Machining

In the case of NC events, the workplane, RTCP, spindle, and coolant are always reset before the NC event. After the NC event, a new approach movement is generated. This includes switching on the spindle and coolant. This is independent of whether the next job after the NC event has the same inclination as the previous one. This is necessary to ensure safe and clear use of the NC event.

OPEN MIND recommendation

OPEN MIND recommends using the NC-Text job option to insert simple M commands (for example, activation of the chip conveyor) or comments into the NC program. We also recommend using the correct VNC variables for commands (including M commands) that result in a movement or changed behavior (for example, activation of RTCP) of the CNC machine.
