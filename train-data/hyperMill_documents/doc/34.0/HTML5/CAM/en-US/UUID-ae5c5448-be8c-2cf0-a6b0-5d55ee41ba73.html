Machine properties / NC file properties

Use the two dialogs to specify which parameters are included in the generation of NC files.

Note that the data entered in the Machine properties dialog is saved as part of the machine definition. You can define additional machine-specific information such as name, postprocessor or origins. When you create a new NC job, all of the properties of the current machine are imported.

You can use the NC-File properties dialog to overwrite specific machine definition properties.

General

Name/ Type: Enter machine designation and type.

Adaptor: Set the machine adaptor.

Only available for postprocessor technology

Postprocessor can use stored solutions: Activated automatically when the postprocessor and machine are coordinated.

Postprocessor

Select the postprocessor file (*.oma file).

Note

OPEN MIND recommends that you manage all postprocessor files in the Global working space.

C:\Users\Public\Documents\OPEN MIND\hyperVIEW

You can no longer manage files in the

C:\ProgramData\OPENMIND\hyperVIEW

folder due to compatibility issues with Windows User Account Control (UAC).

You can also use a postprocessor in a shared network. In this case, you should always save the *.def file as a write-protected file in order to protect it from unwanted changes.

Configure the postprocessor.

Show information about postprocessor.

Copy information on the postprocessor to the Windows clipboard and, if necessary, make it available to your OPEN MIND partner. To do this, paste the information stored in the clipboard into an e-mail.

Model mode: Select the required model mode of the machine simulation.

Modell file

Select a machine model file (*.mmf, *.mmb or *.vmm).

Configure the machine model.

Show information on the machine model.

Copy information on the machine model to the Windows clipboard and, if necessary, make it available to your OPEN MIND partner. To do this, paste the information stored in the clipboard into an e-mail.

Only available for Virtual Machine technology

Use machine adaptor in macros: This function is activated by default and must be deactivated if macros are used for which no adaptor has been defined. The function is deactivated by default when creating a new Virtual Machine.

Virtual Machine: Enable to use a Virtual Machine that has been preconfigured by OPEN MIND. For more information on this, please contact your OPEN MIND partner.

Use: If you want to use the NC Optimizer for this Virtual Machine.

Virtual Machine directory

Select the Virtual Machine directory.

Edit the active configuration.

Show information about the Virtual Machine.

Copy information on the Virtual Machine to the Windows clipboard and, if necessary, make it available to your OPEN MIND partner. To do this, paste the information stored in the clipboard into an e-mail.

Virtual Machine version: The current Virtual Machine version is displayed.

Virtual Machine configuration: Select the configuration of the Virtual Machine.

Fixed with tool database: When the machine is selected in the job list, a check is performed to determine whether the database that is assigned to the VIRTUAL machine in the Machine properties dialog matches the database selected in the project (hyperMILL- Settings - dialog page Database) is selected. If this is not the case, the following message is displayed: Tool database does not match VM, continue? With No another Virtual Machine can be selected in the Machine administration dialog.

Before the NC file is created, it is checked again whether the database selected in Machine properties dialog matches the database selected in the project. If they do not match, a corresponding message is displayed again. If you select No, no NC file is created.

External simulation: VERICUT is available as an external simulation program.

Job mode

One NC file per job: Select this option if a separate NC file is to be generated for each toolpath. Otherwise, all toolpath files will be written to a single NC file during the postprocessor run. The job name or job list name is used here. If a separate NC file is written for each toolpath, the rule for file name generation is applied when the respective NC file is generated.

Compound job as single job: A separate NC file is generated for a compound job.

Rule for file name generation: Any combination of numbers and characters may be entered. The pre-defined parameters for the job list name, job name, job ID and counter are also available.

Start of counter: The generated files are numbered consecutively, beginning with the first number specified (n; n+1; n+2).

Increment to job ID: Amount of incremental job ID increase.

Number format: The %d format corresponds to the printf C++ command.

Numbers

First program number / First block number: For program and block numbering, you can specify the first number as well as the required increment (depending on which postprocessor is selected).

Tool

Type of tool change: Tools can be changed manually or automatically. Tool change position : If the tool is changed manually, you need to enter the corresponding position here.

Go to position before first tool change: The position of the manual tool change is also considered/approached for the first tool.

Time for automatic tool change: Enter the time available for automatic tool changing.

Positions

Start of machining X/Y/Z and End of machining X/Y/Z: If required for control, enter the tool position at the start and end of machining.

4/5 axes

Rotation axis X, Y, Z: Center position of rotation axis in NCS.

Clearance radius / Clearance distance: Define the clearance radius for machines with more than three axes. Define the clearance between the tool and the rotation axis or workpiece (when indexing).

Clearance height: Z position of the tool before tilting.

Axis interpolation: Specify the angle or vector (great circle).

Max. deviation (degree): Enter a value.

Note

The evaluation of this parameter is PP-specific.

Limits

Maximum rapid feedrate / Maximum machining feedrate / Maximal spindle RPM / Maximum file size / Maximum number of NC-Lines: If the toolpath file ( *.pof) contains large values for the above parameters, they are automatically reduced to the maximum values specified here if the corresponding option is selected.

Withdraw and retract movements and machine stop are entered prior to separation.

Cylinder Barrel

Settings in this category are only required in the specific case of 2.5D cylinder barrel machining. This requires a special postprocessor.

Program halt

Enter commands in the NC file that trigger a program halt at critical points.

Under Mode, specify whether these commands should be inserted before or after a tool change.

Commands: Enter the commands you want to insert in the NC file.

NCS transformation

Define a predetermined offset for the NCS. First, the rotation is executed, followed by the displacement. Thus, a displacement refers to the rotated NCS. A warning is issued in this context when loading data in hyperVIEW and during the postprocessor run.

Select the Enabled checkbox to turn the NCS transformation on. X offset moves the NCS in the direction of the X axis, Y offset in the direction of the Y axis and Z offset in the direction of the Z axis. A value must be entered for the required distance. Rotary axis determines the axis of the NCS around which the NCS is to be turned. Rotation angle defines the value for the rotation. It is applied only if the rotation axis has the value x, y or z.

Collision check

Only available in the Machine properties dialog.

Collision check options for the simulation. Select the Enabled checkbox below the required option in order to enable it. Tolerance and color can be specified for each individual option. All other information, such as machine, stock and so on, are taken from the first NC job in the hyperVIEW browser.

Machine simulation

Collision check mode: Only available with a standard postprocessor. Use the Off, On and Head against table options to control collision checking between the machine components for machine models in *.mmb format.

Off: The movements between the machine components are not checked for collisions.

On: The movements between the machine components are checked for collisions.

Head against table Only the movements between the head and the table of the machine are checked for collisions.

Tool spindle alignment

Arbitrary tool spindle alignment: When this option is enabled, the tool spindle angle is not checked during the postprocessor run.

When the option is disabled, the tool spindle alignment is checked during the postprocessor run in relation to the orientation of the X axis of the job frame. If the tool spindle alignment deviates, a message appears, indicating by how many degrees it deviates.

Note

In the context of performing back boring actions, this function is used to check whether the alignment of the tool spindle of turning tools is identical to the alignment of the X axis of the job frame before the tool moves eccentrically into and out of the hole.
