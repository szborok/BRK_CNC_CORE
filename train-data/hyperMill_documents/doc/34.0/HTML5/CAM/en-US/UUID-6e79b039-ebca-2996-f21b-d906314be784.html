3.10. Calculate and analyze toolpaths

Prerequisites, information and options for calculating toolpaths.

Delete, read, update and analyze toolpaths.

Requirements

  1. Correct job sequence: The sequence of jobs in the job list matches the machining order.

  2. Correct job definition: Before calculating a toolpath check jobs that have an error or warning notification. Use the Messages function on the shortcut menu for this. If these are not manually checked, all messages will be displayed prior to the toolpath calculation.

  3. For 3D and 5-axis machining, a milling area must be present in order to calculate a toolpath.

Jobs whose machining ended with an error should only be post processed after an extensive check.

General settings

The system of units that has been set is valid for the toolpath calculation. Changes to the measurement system are not permitted after the jobs, tools and frames have been defined. Relevant settings for calculating toolpaths are specified in the hyperMILL Settings dialog, on the Calculation tab.

Job list-specific settings

The following are defined on a job-specific basis:

  • Name and path of toolpaths

  • NCS (program zero)

  • Original stock (if necessary) and model

  • Postprocessor-specific settings

Note

The NCS must be identical for all jobs within a job list. If the NCS is moved, previously calculated jobs become invalid.

Job-specific settings

The frame is set on a job-specific basis.

The display colors and sizes for the toolpath and collision output can also be set on a job-specific basis.
