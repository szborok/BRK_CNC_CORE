Synchronise data

To synchronize tool data of a supported tool management system with the OPEN MIND tool database, proceed as follows:

Activate the Child database option, if this database should be "read-only". Only the synchronization service can then write to the database, so that accidental changes are prevented.

  1. In the database browser, select the database into which you want to import the data of another system.

  2. Choose Synchronization... in the context menu of the database entry. The OPEN MIND tool database synchronisation service dialog opens.

  3. With Input format optionally define the format of the data to be imported. The neutral data exchange format of OPEN MIND (.xml) is used by default. If Preserve folder structure is activated, the folder structure of the data is not changed during the import. The function is only available for the neutral data exchange format from OPEN MIND.

  		4.

    For Database select the database into which tools from the other system are to be imported, i.e. the database used for synchronization. If the dialog is opened from the tool database, the database for which the dialog was opened is already entered.

  5. For Data exchange folder select the folder where to which the tool management system exports tools.

  6. Control the data exchange in the Service area. The Start button starts the data exchange, to end click on the Stop button. For Log file select a folder in which the log file with information about synchronization is saved and define a name for the log file.

  7. Click Close to close the dialog.

Command line functions for synchronization

The synchronization of the OPEN MIND tool database with one of the supported tool management systems can also be controlled via the command line. To start the synchronization service, call the file omTdbService.exe. This file is located in the OPEN MIND installation directory under Tool Database\[version]. The following functions are available:

/i[nstall]

Installs and starts the service.

/r[emove]

Stops and uninstalls the service.

/h[elp]

Shows this message.

/f[older] P

Absolute path to the folder where the tool management system exports tools to.

/d[atabase] D

Database in which to import tools from the other system.

/t[ype] T

Optional import format type. The following format types are supported:

std: Expects XML files using the OPEN MIND neutral data exchange format.

tdm: Expects files in the format as they are delivered by products of TDM Systems GmbH. If not given, std is assumed.

/p[reserve]

Preserve folder structure. Only supported if format type is std.

/s[lave]

Marks the database as read-only Slave database. Accidental data changes are prevented.

/polling-time T

The service uses folder change notifications to get informed when to start synchronizations. But after some time of inactivity it looks itself at the folder for changes. The time of inactivity can be controlled by T (in seconds - 60 = default).

/reset-slave

Resets the Slave database flag of the given database. This should only be used if the service fails to reset the flag in the database.

Note

Example for a command to install and start the synchronization service:

"C:\Program Files\OPEN MIND\Tool Database\[version]\omTdbService.exe" /i /d "C:\database\my_sql_server_tdb.dns" /f "C:\database\import"

This command line starts the service for the server database, that is specified in my_sql_server_tdb.dns and expects the files of the tool management system underC:\database\import.
