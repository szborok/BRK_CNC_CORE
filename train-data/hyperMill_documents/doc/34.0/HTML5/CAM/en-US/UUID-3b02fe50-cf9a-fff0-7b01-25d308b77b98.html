Define parameters

Input field colors

Black input value: The value is correct.

Red input value: The job can be calculated with this value, however, collisions might occur. Read the warning in the message window. Change the respective values if necessary and check the job in simulation after calculating the toolpath.

Input field values

Values can be entered as follows:

  • Direct entry of required value into input field.

  • Use of a predefined local cycle parameters.

  • Definition of user variables.

  • Definition using arithmetical expressions.

Local parameters and user variables can be combined in a single expression. You may also enter all values directly into the input field using the keyboard.

Local cycle parameters

Depending on the cycle and input dialog, you can also use predefined local cycle parameters.

Click the icon to select a local parameter directly from the shortcut menu (J:). All the entries that you make directly in the input field can be made without the J: prefix (as long as there are no user-defined variables with the same name). No distinction is made between upper and lower case.

Lock parameters

Right-click in the dialog to the right of a parameter and use Lock to lock the parameter or Unlock to unlock a locked parameter again.

Example of use: path compensation

If the tools are set to the delta value = 0 by default in the control units in your production (R=0), lock the parameters for Path compensation so that the toolpath is always output to the tool center point.

User variables

Click the icon (to the right of the input fields) to define a user variable and select the Create user variables option.

The User Variable window opens. Right-click and then select New.... Enter the required data in the Code, Value and Description fields. You can also enter predefined arithmetical expressions in the Value field. Double-click the corresponding line of a user variable to change its definition. The most recently used variables are displayed in the middle third of the shortcut menu (max.5 variables displayed). All data that you enter directly into the input fields has to be entered in exactly the same way as defined in the variable. Upper and lower case is not freely definable.

Using the Import shortcut menu of the User variables dialog, import a list of user-defined variables with Excel. Supported formats:

  • For hyperMILL: *.vst

  • For Excel: Text (Tab delimited) (*.txt) and Formatted Text (Space delimited) (*.prn).

Requirements for importing user variables via an Excel table:

  1. The table must consist of three columns (name, diameter, comment).

  2. In the selection dialog *.* must be set as a file type.

  3. The file to be imported must be available in *.txt or *.prn format.

Arithmetical expressions

The basic arithmetical operations and brackets are allowed as arithmetical expressions.

To define arithmetic expressions, click the icon and define the formula using the permitted formula elements.
