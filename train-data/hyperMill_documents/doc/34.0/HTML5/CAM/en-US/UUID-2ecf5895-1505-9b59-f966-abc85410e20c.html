Contours

Define machining contours, the vertical machining area and cycle-specific contour parameters.

Warning

Ensure that the tool is only in contact with the model geometry on one side (1). The tool diameter must not correspond exactly with the model geometry to be machined (2).

Contour selection

Each cycle allows machining of several pockets, each with several islands.

Geometric information from the Pocket feature can also be used as contours.

The contours must exist as: closed polylines (2D), circles, ellipses or splines. The order of contour selection is arbitrary. Pockets and islands are recognized automatically. All external contours are pocket contours that are machined from the inside. Multiple adjacent pockets can be machined in one single job step.

To manually define the top and bottom, the cycle offers the options Absolute (jobframe), Top of contour, Bottom of contour.

Examples:

  • Different tops: In the following example, for the four adjacent pockets with different tops but the same absolute bottom, take the Bottom (1) directly from the model as an absolute value related to the job frame by clicking the icon and the Top (2) using the option Top of contour and entering the value manually.

  • Island / inner pocket: In the following example, it is an island (A) or inner pocket (B).

    (A) Top and Bottom are defined by selecting the contour at the bottom of the pocket (1). Then take the value for the top directly from the model by point selection and with the help of the Absolute (jobframe) option.

    (B) The top definition is made by selecting the contour at the top of the pocket (1). Then define the bottom using the option Relative to top (2) and manually entering the bottom.

Contour attributes

To define contour attributes (top, bottom), select the machining contour in the contour list. By making multiple selections in this contour list, you can quickly define the same top and bottom values for several contours.

Use the following buttons to change the order of selected contours or delete a contour. (1) To the top, (2) Up, (3) Delete, (4) Down, (5) To the bottom.

Machining of open pockets

To machine an open pocket, link the job with a Pocket feature. The starting point in this process is outside of the pocket, so that the entire machining process for the pocket can be carried out.

Open areas: Define the open areas of the pocket.

Add through three points: Select start point, end point and a further point on the contour.

Add through curves: Select curves.

Note

Updates of pocket contours created by means of parametric V-sketch are adopted during a recalculation in the cycle, however, this does not apply to Open areas.

Plunge points

Use plunge point: For 2D mode (dialog page Strategy), define either a single global plunge point or several plunge points by means of contour.

Plunge point: Define a separate plunge point for each pocket/contour. The global plunge point is supported by the Generic Pocket feature. A plunge point is not possible if the pocket contains open areas.

No plunge point selected: The optimum starting point for machining the selected contour pockets is determined automatically. If it is not possible to reach all machining areas from one plunge point, the program automatically looks for additional plunge points. Rapid infeed between plunge points takes place across the clearance plane.

Plunge point note: Since the user-defined plunge point is not collision checked, you must ensure that collisions will not occur. User-defined plunge point and stock mode are mutually exclusive.

The tool moves exactly to the specified plunge point and then machines from there to the center. The pocket will always be cleared from the inside out.

In stock mode, machining always proceeds from the outside in.

If the plunge point is outside the pocket contour, the tool moves into the contour pocket along the shortest path (G1).

Warning

Danger of collision!

Contours that the tool cannot reach from the user-defined plunge point cannot be machined.

Optimization

Sort contours: Select this option to sort the traversing movements between the pocket contours according to the shortest distance in each case. By default, the traversing movements are performed according to the selection order of the pocket contours. Available for 2D mode, 3D mode and Stock mode as well as for the Use adaptive pocket option and MAXX Machining.
