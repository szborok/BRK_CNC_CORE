Calculate toolpaths

Either directly in the job definition (for an individual job) or in the hyperMILL browser (for an individual job, multiple jobs or all the jobs in a job list).

In the job definition

Calculate: The main purpose of this method of creating a toolpath is to enable a quick check of the work-through of an individual job.

In the hyperMILL browser

Select the required jobs or job list and then select the Calculate function from the shortcut menu.

When calculating/updating one or more linking jobs, the system displays a message asking whether you want to recalculate all sub-jobs. If you choose No, only the linking job, and not its sub-jobs, are recalculated.

Prior to the calculation, the messages stored for the jobs or joblists to be calculated are displayed.

Ignore: Ignore the messages and calculate the job.

Ignore all: Ignore all messages and calculate all jobs.

Cancel: Cancel the calculation.

Edit: Open the dialog (Job/Joblist) and edit the settings.

During the calculation, the message window displays information on the status of the calculation. The calculation can be controlled as follows:

Cancel the toolpath calculation.

Stop the toolpath calculation.

Continue the toolpath calculation.

For some machining cycles, temporary toolpaths are displayed initially that correspond to the calculation sequence. Once the calculation has finished, the calculated toolpaths will be displayed if you have selected the Display toolpaths option in the hyperMILL settings dialog.

If the system identifies collisions while calculating the toolpaths, they are also displayed graphically.

Information on the calculation process is displayed in the window that appears at the same time. In the hyper MILL settings dialog, enable the required option under Message mode on the Application dialog page. In addition, various messages can be viewed in the report file. The report file (*.rep) is stored by default in the Global Working Space.

Tip

If the name of a job list or a job comment is changed after the toolpaths have been calculated, it is not necessary to subsequently recalculate the toolpaths of the job list. hyperMILL automatically adapts the toolpaths accordingly.
