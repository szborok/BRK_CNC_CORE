Frequently asked questions

The first section provides you with additional tips and tricks (divided into subject areas) for the efficient use of hyperMILL.

If you cannot solve your problem by means of the FAQs, please contact the OPEN MIND support or your OPEN MIND partner. In order to be able to process your request quickly and efficiently plus resolve your problem as quickly as possible, we require some information from you, which is summarised in section ???.

hyperMILL installation

Licensing / Registration

hyperMILL

I purchased a hyperMILL license, but the hyperMILL browser cannot be opened and the following message appears:

hyperMILL [version] is not licensed on this computer! Unable to execute the requested command!

Make sure the license is intended for the installed version. Make sure the dongle is installed and recognized correctly. Please find information on activating your hyperMILL license in the installation guide, chapter 8 Activate and Manage OPEN MIND License Manager Products.

OPEN MIND direct interfaces

I have bought a license for the OPEN MIND direct interfaces, but the toolbar for the interfaces is not available.

Check on the Activation code tab of your OPEN MIND License Manager whether you have activated the direct interfaces for your dongle. Please find information on activation in the Activate and Manage OPEN MIND License Manager Products section of your installation guide.

Post Processors

New post processor

I have got a new post processor. How to proceed?

  1. Follow notes

    Please pay attention to the notes that are enclosed with the postprocessor delivery. If necessary, please consult your OPEN MIND partner.

  2. Installation

    For information on installing a postprocessor, see section Set up machine.

  3. Adapt configuration file

    Before using the postprocessor for the first time, please adapt all parameters of the configuration file ( *.def file) belonging to the postprocessor that are especially important for your purposes.

    The adaptation of the configuration file is a continuous process. Depending on the frequency of the usage of the postprocessor with different tool path files, this process can take several months, if necessary.

  4. Testing

    Please test the new postprocessor with typical toolpaths [according to your usual requirements]. Please don't use a current project for this, but (if possible) please use preferably a workpiece, which is not used for commercial purposes.

    If only a commercial workpiece is available, please only use toolpaths that maintain a defined distance to the workpiece for testing.

    First, please test single toolpaths, i.e., single jobs without tool changes. Afterwards, the testing procedure can be enhanced and the single jobs tested before, can be tested with machining priority including tool change.

Post processor update

I have received an update of an post processor. How to proceed?

To implement the current changes of your post processor, please proceed as follows:

  1. Back up

    Please first back up the existing postprocessor files to prevent them from being unintentionally overwritten by the newly provided postprocessor files. Doing so means that the original postprocessor files will be available as backup files further on.

  2. Apply adjustments to update

    Please also include the adjustments made to the postprocessor's configuration file(s) in the postprocessor update. If necessary, please consult your OPEN MIND partner.

    Note

    The configuration file is a text file. You can compare the updated file with the existing file und let display the differences into a textfile editor.

  3. Adjust specified path

    If you saved the updated postprocessor files (oma files) in another directory (different from the directory usually used) or if you renamed the updated file, you also need to adjust the path in the machine administration of hyperVIEW (on the hyperVIEW menu in Edit → Machine administration → New → General → Machine properties → Postprocessor).

    Note

    Please always save the updated files in a separate directory to prevent them from being overwritten by accident. Please inform the administrator of your hyperMILL installation about the updated postprocessor, so that the administrator can use the latest postprocessor file for subsequent installations, for example, after a new hyperMILL release. If the updated postprocessor is used by a number of people in your company, all the people concerned should be informed about every update.

  4. Testing

    Postprocessor updates have to be tested (as is the case with new postprocessors).

Automatic postprocessor updates

Are updates automatically shipped?

No. Updates are neither generated automatically nor shipped. Therefore you should contact your OPEN MIND partner once a year, to find out about the availability of a postprocessor update.

Adjustments to the postprocessor

I want to have a adjustment of the NC output of the post processor? How to proceed?

Please contact your OPEN MIND partner. The information, named in the following is required for handling your request:

  1. Which postprocessor should be updated?

    Please use the function in hyperVIEW Machine properties → Postprocessor → Info.

    You can copy all the information shown in the Postprocessor Information dialog directly to the clipboard and save it to a text file.

  2. Create hvz file (including NC file).

    Please generate an hvz file from all operations (jobs) that you have postprocessed together.

    Please make sure to generate this hvz file after postprocessing in order to ensure that the corresponding NC file exists.

  3. Document desired changes.

    Please document the desired change using a specific hyperMILL project and inform your OPEN MIND partner in detail about all functions you wish to have in the new postprocessor.

    For this purpose, please generate a new NC file and also a text file that describes the desired changes (please use the corresponding machine and/or the CNC control manuals, if necessary).

    Note

    Please note: Adjustments to a postprocessor may incur additional fees! Your OPEN MIND partner will inform you, before expenses are incurred.

  4. CAD file

    In some cases, the CAD file (including job list) from which you generated the hvz file (see section b) may be required. Please also provide this data to your OPEN MIND partner on request.

Postprocessor and new hyperMILL

Do I have to install a new postprocessor after a new hyperMILL release?

OPEN MIND endeavors to ensure compatibility with existing postprocessors with its new releases. However, you can check the behavior of the postprocessors after the introduction of a new hyperMILL release.

We recommend doing this based on the last project that was edited and postprocessed before the new hyperMILL release went live. If the toolpaths are not recalculated in the new hyperMILL release, the new hyperMILL release should generate exactly the same NC programs. Furthermore, OPEN MIND recommends that you check the toolpaths typically used in your applications.

Examples: If you primarily program 2D toolpaths with hyperMILL, you should check some of the 2D toolpaths. If you primarily generate 5X machinings, please compare some 5X toolpaths.

However, if the results deviate from the usual NC output in a new release, please first check if the postprocessor files last provided for hyperMILL were installed by your administrator when the new hyperMILL release was implemented.

Please also contact your OPEN MIND partner and inform him or her about this incompatible behavior following the installation of a new hyperMILL release. Your OPEN MIND partner will analyze the incompatible behavior and provide further information.

Furthermore, after installing a new hyperMILL release, you should check whether a new postprocessor release is also be available.

Postprocessor and new dongle

What is to consider when exchanging the dongle?

If you don't have new post processor files, please contact your OPEN MIND partner and provide all current dongle information. Please note: it may take a number of days to provide you with these files.
