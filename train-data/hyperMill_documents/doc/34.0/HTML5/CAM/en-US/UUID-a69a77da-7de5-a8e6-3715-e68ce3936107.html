Solutions

Preferred direction

For machines with 5-axis kinematics, define the preferred spindle orientation in machining situations with at least two solutions.

Use global definition: Enable to use the default settings of the VIRTUAL Machining postprocessor for creating the NC file. Disable the option to use the settings of the current job (the default settings of the VIRTUAL Machining postprocessor are overwritten in this case). Use Plus or Minus to define the preferred sign for the rotary axis.

Example:

BC kinematics: When Plus (1) is specified as the preferred axial direction, select the one that tilts the component to the viewer.

Force: When the NC Optimizer is activated, the direction of the preferred axis is also taken into account, however, the NC Optimizer may make a different choice for different reasons (axis limits exceeded, collision avoidance). This behavior of the NC Optimizer is disabled by the Force option.

Closest C angle plus / Closest C angle minus: The solution in the head that is closest to the defined C angle is selected. The solution in the head can switch between plus and minus, which keeps the movement of the C axis small.

Additional information: Closest C angle plus / Closest C angle minus

If there are two possible solutions with the same distance to the defined C angle, the desired behavior of the machine can be defined via Closest C angle plus or Closest C angle minus. Here plus and minus refer to the C-axis solution.

Example: BC head/head kinematics, angle C axis = 0 degrees

(1) Solution for Closest C angle plus = B+32.8239 C+79.2792.

(2) Solution for Closest C angle minus = B-32.8239 C-79.2792.

Example: BC head/table kinematics, angle for the C axis = 0

(1) Solution for Closest C angle plus = B+35.6477 C+77.1914.

(2) Solution for Closest C angle minus = B-35.6477 C+282.8086.

Preferred solution C-axis (A/B=0) / Preferred C-axis turning (A/B=0)

Define a manual pre-rotation of the C axis before the start of the job with the Preferred solution C-Axis (A/B = 0)/Preferred solution C-Axis turning (A/B = 0) function. This makes it possible to manually avoid axis limit violations or collision situations.

Example 1: 2D Milling

BC kinematics:

The function Preferred solution C-Axis (A/B = 0) → Angle is set to 180.

This ensures that the C axis is rotated to 180 degrees before the start of the job to stop the axis limits from being exceeded. Tool position inside X axis limit (1), tool position outside X axis limit (2).

Example 2: Milling with rotation axis

The function Preferred solution C-Axis (A/B = 0) → Axis change is set to 90°. This ensures that an XY milling movement is converted into a milling movement with rotation axis and linear axis (e.g. CX).

Control the position of the tool by changing the value for the angle. The basic position 0° refers to the X+ axis of the NCS. The value for the angle is a solid angle (counterclockwise). If the specified value for the milling position leads to a problem, such as an axis limitation, an optimization is automatically carried out by the NC Optimizer.

Requirement: The Use NC optimizer function is enabled in the machine properties dialog.

(1) Axis change = 0°, (2) axis change = 90°

The option Preferred solution C-axis (A/B=0) → Axis change is enabled and the value for the angle is 90°.

The option Preferred solution C-Axis (A/B = 0) → Frame is enabled.

Example 3: Drilling

AC kinematics

The function Preferred solution C-Axis (A/B = 0) → Angle is set to 0 (1).

The function Preferred solution C-Axis (A/B = 0) → Angle is set to 45 (2). This rotates the C axis clockwise by 45 degrees before the job starts.

Example 4: Turning

AC kinematics:

The function Preferred solution C-Axis turning (A/B = 0) → Angle is set to 0 (degrees) (1).

The function Preferred solution C-Axis turning (A/B = 0) → Angle is set to 45 (degrees) (2). This rotates the machining position by 45 degrees.

Frame: The NC output and the C axis position correspond to the frame selected in the job.

Transform polar: Only available for lathes and turn-mill machines. The NC output is via the C-X axis and the corresponding functions in the control.

Axis change: The angle entered defines the position of the milling tool during machining. The NC output is via RTCP and a rotary axis. Output is only possible in the G17 inclination.

Force: When the NC Optimizer is activated, the specified C axis is generally taken into account, however, the NC Optimizer may make a different choice for different reasons (axis limits exceeded, collision avoidance).

This behavior of the NC Optimizer is disabled by the Force option and the C axis is always pre-turned.
