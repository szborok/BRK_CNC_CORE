Application dialog page

Dialogs

Define the dialog control mode (Modal/Modeless) as well as the dialog tabs display mode (text/icons).

Show tooltips: Show the tooltips.

Message mode

Specify the information to be displayed as messages during job definition. These settings do not relate to message windows you call up manually. Each message mode (Severe errors, Errors, Severe warnings, Warnings, Infos) automatically includes the respective next ‘level’.

Model check

Automatic: Changes to the CAD model trigger a consistency check between the defined jobs and the CAD model. If automatic model checking is switched off, it can be performed separately for each job via the job or joblist’s shortcut menu. Regardless of this setting, a model check is always carried out before generating the toolpaths.

Licensing

License status: Shows the status of your hyperMILL licenses.

All products: Locked and available products are displayed.

Locked products: Only locked products are displayed.

Licensed products: Only available (and licensed) products are displayed.

If you work with network licenses, you can reserve specific licenses for your computer when you start hyperMILL. Select the product that you want to lock and enable the Lock on start option.

For more information on network installations, refer to the installation guide for your product.

Show locked products only: When you create a new job, only those products that are available for your computer (locked products) are displayed.

2D compatibility

Limit 2D jobs to NCS: You cannot select frames in 2D jobs. Select this setting if you are only going to program on the NCS (program zero point) and your postprocessor is not suitable for multi-axis indexing.

Default paths

Paths for saving files generated with hyperMILL. Use the following icons to make changes.

Add/remove directory.

Change path.

Global Working Space: Location for storing all data that is generated by hyperMILL (toolpaths, models, databases and so on). Changes are possible only using the hyperMILL SettingsWizard (hmInit.exe).

hyperMILL search path: The directories are set by default but can be changed if required. Once paths have been correctly set here, only the name of each file needs to be entered for all further processes.

Temporary files: Only temporary files needed within the program, including, for example, boundary files ( *.bnd), profile files ( *.prf) and calculation files for individual cycles ( *.omx).

Configuration files: User-specific default settings.

Projects: Files will be stored in this directory if you have selected the Project directories option on the Document dialog page of the Setup hyperMILL dialog.

3DF files: Directory for polyhedron files (mathematical collision models for calculating toolpaths).

Hmrep files: Directory for temporary geometry data.

Stock files: Directory for stock files in *.STL or *.VIS format.

Toolpaths: Directory for toolpaths with information that is required to generate an NC program.

Backup files: Directory for *.hyperMILLbackup files.

Report files: Directory for report files with messages output from the program during toolpath calculation (error messages, warnings and so on).

NC files: Directory for NC files (postprocessor run).

Export- / Import files: Directory for the exported or imported jobs of a job list.

Macros: Directory for macro files.

CPF files: Directory for Customized Process Feature files.

Programming tasks: This is only shown if the corresponding license has been purchased and after launching tire machining.

AddIns: Not used currently.

AUTOMATION Center: Default path to store data generated or used by the AUTOMATION Center, such as fixtures, features, stock models or reports.

Turret setup: Directory for saving exported turret setup.

Performance

Use all CPU resources for cycle computing: The majority of cycles support multi-core processors and perform calculations in parallel. If this option is selected, all of your computer's CPU resources will be used for cycle computing. Please note that the simultaneous use of hyperMILL for interactive work during a calculation may slow down the system. If the option is not selected, a part of your computer's CPU resources will be available during cycle computing for simultaneous hyperMILL use, which means that toolpath calculation may be slowed down as a result.

Backup

The Activate option allows hyperMILL to create backup copies of the CAD model files used ( *.hmc / *.e3). Backup files are created as:

[Original file name]_ [n].hmc.hyperMILL

To open a backup file in hyperMILL, change the file extension to *.hmc / *.e3.

Number of backup copies per document: Specifies the number of backup copies per CAD model file. At present, you can create up to 999 backup copies. Each time you save the model file, a backup copy is created. Once the maximum number of backup copies is reached, the backup copy created first is overwritten.

Save backup in user-defined folder: Backup files are saved in a user-defined directory. This directory is either the directory displayed under Default paths → Backup files or (if the Project directories option has been enabled on the Document dialog page) in the corresponding project directory.

Save backup in same folder as original: Backup files are saved in the same folder as the original file.
