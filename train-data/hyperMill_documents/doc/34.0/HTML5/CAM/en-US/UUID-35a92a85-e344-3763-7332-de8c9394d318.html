Setup dialog page

Limit the search using filter criteria. Define comparison parameters for the search. Define conditions for accessibility of pockets and the creation of feature lists.

Pocket filters

Tool diameter: Limit the pocket size using the tool diameter. Only pockets matching the following criteria will be recognized: the upper diameter should not fit in the pocket, while the lower diameter should still fit.

Circular pocket limit: This option can be used to prevent holes from being recognized as circular pockets by defining a diameter limitation. Only pockets that fall within the defined limitation are recognized.

Chamfer height / Fillet radius: Height of the chamfer or radius of the fillet between the bottom surface and the pocket wall.

Chamfer angle: Lower / upper limit for the chamfer angle between the bottom surface and the pocket wall.

Allowed wall types: Specify which wall type is to be recognized. The Perpendicular option corresponds to the Z axis of the frame that is defined on the Pocket example dialog page.

Combine pockets: If the Suppress original pockets option is enabled, the original pockets will be deleted after combining.

Pocket comparison parameter

These two parameters relate to the comparison between the sample pocket and the pockets that are to be recognized (see Comparison dialog page).

Scale limit: The minimum / maximum scale limit defines the recognition of pockets in the model on the basis of characteristic geometric parameters.

Comparison tolerance: The degree of accuracy with which the characteristic geometric parameters (see Scale limit) of one or more sample pocket(s) are compared with the pockets to be recognized.

Pocket Recognition can be restricted to certain pockets through the Pocket filter using geometric features.

All pocket filters are in effect when the sample pocket is defined. If they are activated, the options under Accessibility check are effective when searching for comparison pockets.

Accessibility

Since not all pockets are directly accessible and can be processed, conditions for the accessibility of a pocket are defined here.

Accessibility check: the accessibility is checked in the direction of the surface normals of the respective bottom surfaces, and in the opposite direction (1).

Approach length: defines the minimum distance between a potential bottom plane and another surface that may prevent access to the bottom plane (2).

Polyhedron offset: Accessibility of the bottom plane (pocket bottom) is determined on the basis of this contour offset over an extrusion surface. This should always be larger than the value for polyhedron tolerance (3).

Polyhedron tolerance: accuracy of the polyhedron representation for the accessibility check (faceting of the model).

All recognized pockets are transferred as Generic Pocket feature (close/open) to a feature list. Pockets with the same Z direction are combined into one list (frame). The colors of the feature lists that you create are defined in the hyperMILL settings (see dialog page Document â†’ Colors).

Feature list generation

Use feature list: if this option is enabled, the pockets are saved as features in the existing feature lists. Feature assignment is in reference to the frames assigned to feature lists. Feature lists that do not contain frames are ignored.

Use color table: When reading feature information, pocket bottoms with a specific color value (RGB value) are automatically recognized and the respective features are assigned to a corresponding feature class. For this purpose, both the color value and the name of the feature class are defined in an XML file. The name and location of the XML file can be selected freely.

Example:

<Color_Table>

<Color_Machining_Definition
R="214" G="214" B="214"  IsPocketBottom="1" FeatureClass="Special
Pocket"/>

</Color_Table>

To read this information, activate the Use color table option, select the corresponding file (format: *.xml) and confirm your entries. The Pocket feature created contains the designation of the feature class in the feature name.

Example:

Generische Tasche <Geschlossen R10 Spezial-Feature>.

The feature class is displayed in the feature editor (section: General).

Use all colors: Even pocket bottoms whose colors are not contained in the XML file will be recognized. The recognized pockets are, however, not assigned to a feature class.

If the function is disabled, only pocket bottoms that correspond to a color in the XML file will be recognized.

XML files can also be used in the Feature Mapping area to read in feature information.
