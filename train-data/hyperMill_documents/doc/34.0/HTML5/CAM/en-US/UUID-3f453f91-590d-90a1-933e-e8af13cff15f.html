The macro database

A macro database is created in the CONFIGURATION Center. For information, see the installation guide.

You can manage and configure your macros in the macro database. The following functions are available:

  • Save database

  • Import / export macros

  • Edit feature and job parameters later on

  • Automatic macro search

  • Optimise macros

  • Specify rules for machining modes

Note

Multi-user mode is not supported by the OPEN MIND macro database. This also applies to macro databases with SQL Server technology.

To open a database:

  1. In the hyperMILL browser, go to the Macros tab.

  2. Right-click and select View Macro DB.

User interface

The user interface of the macro database consists of the following elements:

(1) Display of the defined macro groups.

(2) List of the macros contained in the macro groups.

(3) List of macro jobs linked to a selected macro.

(4) When the Test mode option is enabled: List of features that can be processed with a selected macro.

Macro groups

Control the display of the macros in a macro group by activating/deactivating a macro group. The Number column displays the number of macros contained in a macro group.

Macros

The list view displays the macro name as an abbreviation of the feature or job parameters with the name affix ( F) for feature or ( J) for job.

Macro jobs

The list displays only the macro jobs that are contained in the macro that is selected in the upper window of the database.

Save macros

Click Save to save a newly created or updated database. The macros contained in the database are saved in *.mdb format in the following directory:

C:\Users\Public\Documents\OPEN MIND\mac\

Prioritizing macros

If you are working with several macros for a feature, you can use the Priority (0-9) function to prioritize the macros from 0 to 9 and so define which macro should be used preferentially. Here, 0 has the lowest and 9 the highest priority.

Import / export macros

You can export single macros or all the macros in a database, or you can import macros into the existing database from another database.

To export all the macros of a macro database:

  1. Click Export all and enter a file name or

  2. click File and select an existing file in *.omx format (OPEN MIND Exchange Format).

To export individual macros:

  1. Highlight the respective macros in the right-hand window and click Export selected.

  2. Click File and select an existing file in *.omx format (OPEN MIND Exchange Format).

To import macros from another database:

  1. Click Import â†’ macros.

  2. Select the respective file in *.omx format (OPEN MIND Exchange Format).

Edit macro parameters

You can change the names of macro groups, job parameters and macro parameters in the macro database. This way, you can react quickly and flexibly to changes in the conditions.

Macro group: To change the name of a macro group, double-click the corresponding list entry.

Macro parameter: When you double-click the macro entry in the list, the macro's main parameters are displayed in a separate overview window. Double-click the feature parameters/values in this window to edit them.

Job parameters: To change job parameters, highlight the required macro job in the bottom window of the database and click Edit or double-click the corresponding list entry.

Confirm your changes and close the dialog.

Note

Changes do not take effect until you have clicked Save to update the changes in the database. Changes to macros in the database have no effect on macros already applied in the job browser or macro browser.

Conversely, editing the applied macros has no effect on the macros stored in the database.

Create new macro group

To create a new macro group, right-click the left window and select New.

Move macro

To move a macro to another macro group, right-click the macro you want to move in the right window, select Move to...> and select the macro group to which you want to move the macro.

Delete macro group

To delete a macro group, right-click the macro group you want to delete in the left window and select Delete.

Delete macros

To delete a macro, right-click the macro you want to delete in the right window and select Delete.

Copy macros

To copy a macro, right-click the macro you want to copy in the right window and select Copy.

Copy and paste macro jobs

Use the Job/ Copy all jobs to clipboard function on the shortcut menu of the macro job to copy one or all macro jobs from a macro. Use the Paste job / Paste all jobs function on the shortcut menu of the macro job to paste the copied job/jobs into another macro.

Link a macro job to a stock model

For machining operations with a stock chain (resulting stock), subsequently link a macro job to a stock model if, for example, in a macro

  1. the 3D Z Level Shape Finishing macro jobs are used (as here the stock model is not saved when the macro is created) or

  2. the 3D Optimized Roughing macro jobs are used and a new job was subsequently programmed.

To link a macro job to a stock model, use the following function in the shortcut menu of the previous macro job: Link generated stock to the next job.

Important

The Link generated stock to the next job function is only available for jobs that can create resulting stock.

Tools in the macro database

The Tool data column in the list view indicates how the used tool was created. A distinction is made between the following:

Macro: The tool was defined directly in the macro (or job) without a search of the tool database.

DB: the tool used for the job is taken from the tool database (numbered NC tool).

DB+Auto: Only applies to 2D drilling jobs. Like DB + automatic tool search, the Tool length is taken into account. All of the tools suggested are collision-free.

DB+Auto+: Only applies to 2D drilling jobs. Like DB + automatic tool search, the Tool length and Tool diameter are taken into account. All of the tools suggested are collision-free.

Automatic macro search

During an automatic macro search, hyperMILL searches the database for macros that match the selected feature and then applies these.

Macro optimization

Jobs that can be machined using the same tool are processed in sequence (tool change optimization).
