Tables, rows and columns

Tables are an important visible element for the display of values in a report. Tables are suitable to graphically structure the overall output of a report, as well as to list the individual values of a dataset.

Tables ( table) are divided into an optional header area ( thead) and one or more table body areas ( tbody).

The header area can be used to e.g. display the headings of a tabular list, while the values can be specified as a placeholder using a loop ( xsl:for-each) in the rows ( tr) and columns ( td) of a table body.

In this way, you can generate a tabular list of the used tools with the respective tool properties.

Another function of tables is to specify the display areas inside a page. To do so, the dimensions of the columns are specified explicitly, e.g. a table with a width of 100% can be divided into three columns with widths of 25%, 50% and 25%. It is alternatively possible to specify widths in pixels using the abbreviation px. In this way, the user can divide the page into several display areas and position images or text ranges as needed. Further structure elements can then be hierarchically created inside the display areas.
