Job Optimization

Note

Optimization of manually created jobs as well as macro jobs with the aim of reducing the number of tool changes and shortening machine running times.

Optimization of manually created jobs

To optimize manually created jobs, use the Optimization (only compound jobs) function on the job list’s shortcut menu.

Compound jobs

Group manually created jobs of a job list into compound jobs.

Note

Grouping is also available for macro jobs.

The No optimization option is enabled by default for manually created jobs. The job steps in the job list are grouped into compound jobs either by tool or by machining step. Only directly consecutive jobs for which the tool or the job type remains the same are combined. There is no change in the job sequence.

Example – Group by tool:

No compound jobs: No compound jobs are created.

Group by tool: Jobs with the same tool are combined in a compound job. The name of the generated compound job is composed as follows: [Tool number / Tool diameter / Tool name].

Group by process step: Job steps of the same job type (same job name) are combined in a compound job. The name of the generated compound job is composed as follows: [Cycle name].

Optimization of macro jobs

To optimize macro jobs, use the Optimization (only compound jobs) function on the job list’s shortcut menu.

Optimization

Optimize macro jobs. Manually created jobs are not optimized.

No optimization: No jobs are optimized. The sequence of all job steps remains the same.

Optimization with job grouping: Macro jobs are sorted taking into account the tool sequence within macros. Jobs with different features but the same tool and the same job parameters are combined into one job step.

Optimization without job grouping: Macro jobs are sorted taking into account the tool sequence within macros. Identical job steps with the same tool are not combined into one job step. The job steps are sorted in the order of the tool calls in macros.

Manually created jobs, which are located between macro jobs, remain at their positions in the job list.

Tool Sorting

Optimize macro jobs. Manually created jobs are not optimized. Requirement: The option Optimization → Optimization with job grouping or Optimization → Optimization without job grouping is enabled.

No sorting: No change in the sequence of tool calls.

Note

With ascending and descending sorting, the logical order of the tools within the macro has top priority, which means that sorting may not always be maintained correctly or the same tool may be changed twice.

Sort by ascending diameter: Taking into account the macro order, the tools used are sorted in ascending order by diameter. Example: In a component with holes D6, D8, D9 and M10, the holes D6 and D8 are drilled first, then the core hole for M10 with D8.5 and finally the hole D9. The order within a macro is not changed.

Sort by descending diameter: Taking into account the order within a macro, the tools used are sorted in descending order by diameter.

Example: In a component with holes D6, D8, D9 and M10, first the hole for D9 is created, then the core hole M10 with D8.5 and finally the holes D8 and D6. The order within a macro is not changed.
