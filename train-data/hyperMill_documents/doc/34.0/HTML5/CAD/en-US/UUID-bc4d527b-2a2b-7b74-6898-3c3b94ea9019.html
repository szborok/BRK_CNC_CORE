19.1. Python script support

For complex requirements, an externally generated Python script can be executed in the Python programming language.

Python tab

PYTHONToolkit

2024 | Update 2

Python installation

A PYTHON Toolkit is delivered and licensed as a separate installation. It is embedded in hyperMILL® and customized. Embedded means that only a selected set of modules is installed. The adjustments mainly concern data protection and data security. Additional modules can be installed by an administrator.

Note

Install modules for the local Python installation (for standard installation of hyperMILL®) in

C:\Program Files\OPEN MIND\Shared\[VERSION]\python with

python -m pip install tkinter --user --trusted-host pypi.org --trusted-host files.pythonhosted.org

files are installed in the following folder and can also be copied locally to other computers:

C:\Users\[USER]\AppData\Roaming\Python\Python[VERSION]\site-packages

Python source code editor

A Python source code editor can be installed on the computer. The Visual Studio Code software, for example, is recommended for creating, editing, and debugging. The software is free and offers an ideal platform for Python. The advantage of the software is that it provides code suggestions and explanations when code is written and in debug mode. Neither of these is available in a text editor.

Tip

It is recommended to use automatic saving, switch on in the software Visual Studio Code in File → Automatic saving.

Debugging in the Python source code editor

The Python debugger can be executed directly in Visual Studio Code, to do this, press F5. To use the new command, you must have installed the extension for Visual Studio Code. The extension is located in C:\Program Files\OPEN MIND\Shared\[VERSION]\python. To install it, open Visual Studio Code and click the button with three dots on the “Extensions” menu. Then click “Install from VSIX.”

Reload OM library each time a Python script is executed

The om module is loaded into memory the first time a Python code is executed. This is beneficial for performance, but inconvenient when carrying out development work. To test changes to the module, the software must be restarted each time.

The environment variable HC_RELOAD_PYTHON_LIBRARY, which is set to 1, can be used to activate the loading of the om module each time a Python code is executed, even in release mode.

Python tab

Workspaces

Active workspaces are displayed as tabs. If several workspaces are available, select a workspace. Define the root folders for the workspaces in the Python options. Not all commands are available in all workspaces.

Scripts: Edit and manage the scripts for the active projects in your own user workspace.

Library: The library is a collection of ready-made Python blocks that are intended to provide specific commands. Instead of creating these commands again, reuse these blocks in a Python project using the import command. Edit and manage the scripts for these blocks in the library workspace.

Menu

Open the options in File → Options → Python options.

Open a command prompt using the cmd.exe software program.

Open Python documentation.

Upper section – Python script list

The Python scripts of the workspace are listed.

In the Search line at the top of the tree structure, search for text within this structure. Enter the text to be searched for. Suitable results are displayed as the user is typing. Confirm the entry by pressing the ENTER key. The search history will be saved temporarily. Separate text fragments with spaces to display several different contents at the same time. The search is case insensitive.

Clear: Remove the search string again. To do this, right-click the line to display the context menu: Select the command there.

The context menu contains the following commands for a workspace or folder:

Create folder: Create a workspace for Python scripts, that is, create a folder as the root folder for the scripts. A subfolder is created for each Python script. You can also manage the folders and Python scripts directly in the Windows File Explorer. You can create any number of subfolders. The icon is the same as the one in Windows File Explorer.

To rename, double-click the folder name.

Create library: Create a new block of the Python script library (in the Library workspace).

Create project: Create a new Python script project.

A Python script includes a folder, the Python script itself with the same name, and a “.vscode” subfolder with the files “launch.json” and “settings.json” that are required for the Python source code editor. If the .Code folder exists, the icon changes to Python and can be executed with the Python source code editor.

To rename, double-click the Python script name.

The context menu contains the following commands for a Python script:

Run: Python scripts can be started from the Python tab, but not from the Python source code editor. Select a Python script and click Run. Messages about the success or failure of executing a Python script appear in the lower section.

It is possible to define in the Python script whether dialogs should be displayed or executed without dialog feedback.

Redirects the standard output of Python to the user interface window when the code is executed from the user interface, while the standard output is redirected to the output window when executed from code.

Open in Visual Studio Code: Open the selected Python script for editing in the Python source code editor. The OPEN MIND Python modules are already defined in the stored template. If the OPEN MIND Python modules are colored yellow, Python is not connected to hyperMILL® Check the connection to hyperMILL®. If necessary, use a different Python source code editor.

Tip

It is advisable to create a backup copy of the folders on a regular basis.

Search for an error in the Python script. To do this, select the Python script and select the Open in Visual Studio Code command on the context menu. The Python source code editor starts the Python script in debug mode. Set breakpoints in the Python script. Proceed step by step up to these points.

The software is locked during debugging. Only one Python script should be started for debugging. To be able to work in the software again or to debug another Python script, select the “Disconnect” button in the Python source code editor. Do not close or exit the Python source code editor in this state.

Example 28. Procedure in the Python source code editor Visual Studio Code

  1. Set a breakpoint.

  2. Select Debug

  3. Select “Remote Attached”

  4. Start debug: Only the commands F5 “Continue” and SHIFT+F5 “Unlink” are available.

    To exit, either run to the end of the Python script or remove the link. Only in these two states is hyperMILL® released again.



Note

If the Python source code editor is accidentally closed at this point, hyperMILL® remains locked. To fix this, simply reopen the Python source code editor with the Python script and then remove the link.

Prepare use – create menu entry for a Python script

A Python script does not necessarily have to be started from the Python tab. It can also be integrated and started directly on a menu, context menu, or toolbar.

The *.hcconfig.xml file extension is mandatory.

Example 29. Example of a Python script

Create a file C:\Users\[USER]\AppData\Roaming\OPEN MIND\hyperCAD-S\[VERSION]\hcconfig\python_example.hcconfig.xml.

The arguments are optional and begin with the symbol “bar” |.

<?xml version="1.0" encoding="UTF-8" standalone="no" ?>
<Configuration>
	<Commands FormatVersion="1">
		<Command Name="PyArgument" 
		Menu="Python" 
		MenuText="Argument from Python" 
		MinimalPreselection="true" 
		Icon="%MYDIR%/files/images/python_logo.png" 
		Run="RunPython">C:\python_scripts\arguments.py |Length 12.34 draft 12.32
		</Command>
	</Commands>
</Configuration>
#arguments.py
import sys

print("Number of arguments:", len(sys.argv), 'arguments.')
print("Argument List:", str(sys.argv))
print("The script has the name %s" % (sys.argv[0]))

Start Python script

A Python script can be integrated and started directly from a menu, context menu, or toolbar. This means that it must always be started from the hyperMILL® user interface and not from the Python source code editor. It only works in this one direction.

Example 31. Hello World example
if __name__ == "__main__":
    print("This line will be printed.")

The line is output on the Output tab.



Use script as a command

Use a script like a command in the software. Select an option in the context-sensitive menu of a script:

  • Enable command: Enable the use of the script as a command in the software.

  • Edit command: Change the use of the script.

  • Edit command: Switch off the use of the script as a command in the software. First, remove the command from the toolbar.

Command from script

Administrate a command based on a script.

In the Python tab in the context menu of a script via Enable command and Edit command.
