JT-Open file format

Read or save *.jt JT-Open file.

Table 45. Options for opening data via a JT-Open interface

Property

Description

Filter entities

Include entity types and information specifically during import.

Solid conversion mode

Break solids into faces: Solids are broken into faces and faces are transferred. The original layer assignment is retained.

Collect solid faces into groups: The original layer assignment is retained.

Collect solid faces into layers: The original layer assignment is lost when combining on layers.

Create solid: Solids are created again for solids that exist in the original system. The original layer assignment is retained.

In ...\bin\converters\cad interfaces\hcntCR_HE.ini, the value sld_tol_sf can be changed in order to use a different tolerance when closing a solid in relation to the precision of the entity.

Entity conversion modes

Convert hidden entities: Include hidden entities during import.

Create 3D boundaries: Generate curve entities from face boundaries.

Convert all faces to NURBS: All analytical faces are converted to NURBS geometry.

Split closed faces: Select whether closed faces should be divided into two faces or imported as a single face.

Merge meshes: When selecting No, no merging will take place; choosing Merge to one activates merging and creates an overall mesh; choosing Merge by colors merges meshes based on their color. Multiple colors will result in multiple meshes.

Start hyperMILL feature recognition after model import: Feature recognition is performed automatically directly after importing the data.

Model conversion mode

Create model structure: Generate the model structure of the model again. The result may differ from the original model structure.

PMI font

Replace font: Select whether the font should be replaced.

Font: Select a font.

File

Automatic saving after opening: After the import process, automatically save the document with the original file name.

Conversion tolerance

Entity precision: Points within the specified tolerance are handled as a single point. This is a purely mathematical issue and does not concern “point” geometric entities.

Layer assignment

Starting layer for faces: Specify starting layer for combining on layers during the Solid conversion of faces.

Continue layer count: When importing another model into the same document, the layer count is continued and does not start again at the defined starting layer.

Set all layers non-selectable: The entities of all imported layers are set to non-selectable.

By setting all layers to non-selectable directly before opening the file, the performance of handling imported data is improved. Individual layers can then be activated later as required for specific processes.

Log file

Log file mode:

  • Same as input file: Save the log file with the file name of the model in the same folder as the model file.

  • Select path: Select another folder for the log file.

  • Select file: Specify a different file name for the log file.

  • None: Do not generate a log file.

Log file path: Specify a path for the log file.

Log file name: Specify a file name for the log file.

Show log file after import: If a log file is generated, automatically display the file after import.

Color conversion mode

Create colors: Evaluate color and create new color.

Map colors: Evaluate color and assign most similar existing color.

Healing

Enable healing: Repair defective solids (the Create solid option is enabled) and defective faces (the Filter entities → Faces option is enabled). The repair is only carried out for things that can be repaired.

Remove faces less / equal to area (mm² / inch²): Discard faces with an area less than or equal to that specified during import. If 0 is entered, no faces are discarded.

Remove faces with width less / equal (mm/inch): Enter a value for skipping Sliver faces during import – these are line-shaped faces that have no impact on the model geometry. Entering 0 means that all line-shaped faces are accepted.

Advanced

Optimize edges for solid creation: Advanced optimization calculations for generating closed solids.

Template

Merge: Insert CAD data to be imported into ahyperMILL® document template.

hyperMILL document template name (*.hmct): Select document template.



The settings are stored in the file cadifoptjtovalues.xml.

Table 46. Options for saving a JT-Open file

Property

Description

Save hidden entities

You can either save or ignore entities that are identified as hidden.

Conversion tolerance

Define the permissible tolerance for the conversion of entities to an JT-Open entity type.

Keep faces orientation

Optionally retain the existing faces orientation or disregard it.



The settings are stored in the file hewritersettingsjtovalues.xml.

To save and restore settings, right-click to open a context menu containing the following commands:

Table 47. Commands on the context menu:

Command

Explanation

Collapse all

Fold structure tree.

Expand all

Unfold structure tree.

Save all settings

Save the modified settings.

Load all settings

Restore the last saved settings

Reset all settings

Restore the installation settings.

Copy to clipboard

Copy the settings of all sections to the clipboard as text in the form ‘name = value’.

Copy cell to clipboard

Copy the text or value of the selected area to the clipboard.



All settings are stored in files in XML format. For further information about this and about standard settings for multiple users, please refer to the ‘Company-wide default settings’ section.

The commands at the bottom edge of the dialog can be used to save and load all options and settings:

Reset: Restores the factory system settings.

Save settings: Save all settings locally.

Load settings: The local user settings are loaded and updated with the company settings.

If several values are available for selection for an option, click the selection with the left mouse button and simply scroll through the values using the scroll wheel.
