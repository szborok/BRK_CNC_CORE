16.2. Electrode options

Select default settings for electrodes.

File → Options → Electrode options

Electrode

Select default settings for designing electrodes, CAM programming for electrode production, and erosion using electrodes.

In the Search line at the top of the tree structure, search for text within this structure. Enter the text to be searched for. Suitable results are displayed as the user is typing. Confirm the entry by pressing the ENTER key. The search history will be saved temporarily. Separate text fragments with spaces to display several different contents at the same time. The search is case insensitive.

Clear: Remove the search string again. To do this, right-click the line to display the context menu: Select the command there.

The commands at the bottom edge of the dialog can be used to save and load all options and settings:

Reset: Restores the factory system settings.

Save settings: Save all settings locally.

Load settings: The local user settings are loaded and updated with the company settings.

If several values are available for selection for an option, click the selection with the left mouse button and simply scroll through the values using the scroll wheel.

To save and restore settings, right-click to open a context menu containing the following commands:

This option can be used for documents saved as of version 2023.2 | Update 3.

The electrode properties are always saved using the commands Save, Save as and Save selection. That being said, the electrode properties are only loaded using the commands Open, New, and New from template, and not using the Merge command or when providing them internally; for example, holder data or machine data. The reason for this is that the electrode options are a single instance that apply to all documents. This means that when a document is loaded, the existing electrode parameters are overwritten. If the default electrode parameters are required, the electrode properties must be reset or reloaded in the Electrode options dialogue. Put briefly, the last one wins.

Geometry

Prepare

Closing holes from single faces: Features such as holes and breakthroughs inside a Domain (with a single continuous face boundary) can be closed automatically. Switch off the option for eroding lettering, for example.

Extensions

Min. tangent extension: Select the minimum length of a tangent extension at the edges of the erosion faces. A tangent extension is not possible if the geometry crosses over from the erosion area into the remaining workpiece geometry as a curve. No extension is generated if 0 is specified.

Min. linear extension: Select the minimum length of a linear extension at the edges of the erosion faces or the tangential extension. Use collision avoidance and a distance for collision avoidance to increase the length. No extension is generated if 0 is specified.

Length for stabilization extension: To stabilize an electrode, add a stabilization extension by entering a length. No extension is generated if 0 is specified.

Angle for stabilization extension: Enter an angle for the stabilization extension.

Min. block height: Enter a minimum length for the block .

The length of the block can change by adjusting to the rounded position of the electrode holder – if a min. block height not equal to 0 is defined.

If the block height is 0, the correction for rounding the target position is applied to the linear extension. The remaining block height or clamping depth for clamping in the electrode holder remains the same. The reduction is automatically deactivated. If the reference position of the electrode is set to the reduction bottom edge or the electrode holder, the reference position is automatically corrected to the coupling of the electrode holder to the machine axis if the block length is changed to 0.

The length may be increased by reducing the remaining material at the tip of the electrode.

Minimum block offset: Necessary minimum offset to be maintained at the transition between electrode geometry and electrode block or side electrode joint .

C angle

Optimized C angle: The electrode holder is automatically rotated so that fits to the base of the electrode in a way that saves as much material as possible .

C angle: Manually turn the electrode holder without activating the Optimized C angle option. Define a C angle within the range of possible C angles .

Side

Create a side electrode for an erosion area that cannot be reached by vertical erosion (in Z direction). Erode in XZ, YZ or XYZ direction with a side electrode.

Enable creating a side electrode using the Side electrode option.

Control the size of the joint using Joint angle , Min. lateral joint length and Min. vertical joint length . The Optimized joint angle is the half of the angle resulting from the erosion direction and the Z direction of the reference system. The maximum possible joint angle is shown on the Info tab.

Specify the length for the Lateral path. The minimum necessary length is shown in the Info tab. The start position results from the Lateral path plus the Distance for collision avoidance. The Optimized lateral path option automatically calculates a suggestion for the length of the lateral movement.

A rotational electrode cannot be created if a side electrode is to be created.

Rotational

Rotate electrodes to combine roughing and finishing electrodes for an electrode holder.

Use the Rotational electrodes option to switch from single electrode to rotational electrode . Use the Number of copies option to specify how many electrodes are to be created, for example, two for a roughing and a finishing electrode.

The Angle of copies option defines by how much the electrodes continue to be rotated, for example, by 180° if the roughing and the finishing electrode are to lie opposite each other. The value for rotational electrode is permitted in the minus range, that is for clockwise rotation.

The Distance to the rotation axis option specifies the distance between the individual electrodes. The Direction of the rotation axis option specifies whether the distance refers to the X or Y axis of the reference system. Collision warnings for collisions with the workpiece are output on the Info tab.

Target position

Target position: The target position in relation to the electrode reference position.

Automatic target position: Automatically calculate the target position.

To round: Correct the position of an electrode holder, together with the block and the reduction, for the erosion geometry, so as to generate a ‘round’ value for the target position X, Y, Z. For a ‘grid width’ of 1, this will result in a position of 30 instead of 30.234, for example .

Select a value between 0 and 10 for this grid of possible positions . Value 0 does not change the target position .

The correction takes place within the scope defined by the block size in X and Y and the size of the raw material in Z. The height of the block changes. Correction is not possible if the electrode reference position is on the top face or the electrode tip.

Coordinates: Shows the found coordinates of the target position in relation to the reference position of an electrode. If the Automatic target position is switched off for a user-defined electrode, the electrode holder can be moved indefinitely in the XY plane while the raw material retains its centered position relative to the selected faces. With the Simplified mode, it is also possible to move the electrode holder in the Z axis lower than the highest plane of the raw material. An electrode holder for side electrodes can therefore be placed lower down.

These two options are not available in the Create command. If the Automatic target position is switched off there, the electrode holder can only be moved in the XY plane as far as the raw material covers the electrode. Here the raw material follows the movements of the electrode holder. In this case, it is not possible to move in the Z axis lower than the highest plane of the raw material.

Collision avoidance

Collision avoidance: Different parts of an electrode are checked for collisions. A collision is detected if the areas checked against one another plus the safety distance intersect. If possible, the linear extension or block is increased until the collision no longer occurs. If this option is disabled, the collision will be displayed.

For a side electrode, the joint is also checked for collisions. The system also checks if the electrode can move from the clearance position to the start position without a collision. The Info tab provides suggestions on how to prevent a collision between the joint and model geometry.

Distance for collision avoidance: A collision is also detected if the areas checked against one another lie within the specified safety distance .

Machine head and electrode holders consisting of multiple solids are partially checked for collisions for each solid with a separate bounding box. The collision check uses the same safety distance for the machine head as is used for the electrode holder and block.

Raw material

Z length: Control the overhang of a raw material. The overhang is the length of a raw material that is not needed. When the Database height option is used, the complete overhang is in front of the contact face. When the Database height optimized with offset option is used, the minimum required offset is maintained in front of the contact face. The remaining overhang is added to the block height. When the Electrode height with offset option is used, the Z length of the raw material results from the electrode and the minimum offset.

Min. offset: Enter a value for a minimum required offset. The minimum offset is only maintained strictly if the To round option of the Target position is set to 0.

The length of a raw material that is not needed is output as information on the Info tab. If the overhang is smaller than the minimum offset set, the overhang is retained. If a larger value is required, the length must be added to the tangential extension or to the block height so that the next longer raw material is selected from the electrode holder/raw material library.

Example 17. 

If, for example, there is only an overhang of 0.8 mm left in the raw material of Z length 40 mm that is determined in the electrode holder/raw material library, but the required minimum offset is 1.0 mm, the electrode must be extended so that a raw material of length 50 mm is determined. This way, a required minimum offset of 1.0 mm can be maintained.



Technology

Define the following options for each processing step from V for a roughing electrode to VVVV for a polished electrode. The assignment must be made correctly. For example, assigning pre-finishing as finishing results in incorrect EDM settings.

  • Electrode material: Select a material for the electrode from the drop-down menu. Displays all materials that are entered as electrode materials in the XML file for materials selected in the XML file for materials option. Also see User-defined material list.

  • Number of electrodes: Select the number of identical electrodes to be produced.

  • Spark gap for single: Enter the width as a negative value. Consider the width as an undersize during CAM programming.

  • Spark gaps for rotational: Enter the widths from V for roughing to VVVV for fine finishing as negative values and separated by a semicolon. Consider the widths as an undersize during CAM programming.

  • Generator program: Select the required generator program. Displays all generator programs that are entered in the XML file for generator programs selected in the XML file for generator program option. Also see Preparing generator programs.

Document

Enter details about the electrode in the title block of the documentation for an individual electrode using the Electrodes → Print command.

For Creation date and Modification date, variables can be entered as wildcard characters for the date and time.

Variables for date and time

The following variables can be entered as wildcard characters.

%date%

Enter the current date automatically based on the date of the local operating system and write in short format, for example 10/04/2008.

%shortdate%

The date is formatted in the same way for %date%.

%longdate%

Enter the current date automatically based on the date of the local operating system and write in long format, for example Thursday, April 10, 2008.

%time%

Enter the current time automatically based on the time of the local operating system and write in short format, for example 06:30.

%shorttime%

The time is formatted in the same way for %time%.

%longtime%

Enter the current time automatically based on the time of the local operating system and write in long format, for example 06:30:00.

Variables for document

The following variables can be entered as wildcard characters.

%documentname%

The name of the document. Without file extension.

When creating the electrode, they are replaced by applicable information, for example, enter the corresponding date and, if required, the corresponding time in the electrode TAGs T_CREATION_DATE and T_MODIFICATION_DATE.

Reference

Measuring reduction depth: For the reduction, select a depth by which the block should be reduced .

Measuring reduction offset: Select a value to reduce the reduction in relation to the block .

Measuring marker type: Select a marker type, e.g., radius or chamfer.

Measuring marker value: Enter the width of the chamfer at 45° or the radius of the arc .

Electrode name: Default setting for the electrode name and the milling project derived from it.

Reference system name: Default setting for the name of the workplane for the positional reference between the electrode and workpiece.

Separator in names: Default setting for a separator in names, for example, electrode names.

Name starting value: Enter a starting value for the automatic number increment in electrode names, for example, 10, 100 or 1000.

Counter for names: Select the counter increment for the sequential number, e.g., for placing each electrode on a separate layer.

Measuring marker position: Select the position of the measuring marker. Orientation is handled based on the principal alignment (X or Y axis in the direction of the mount) selected in the “Blockdataeditor” program .

Electrode reference position: Select the position of the electrode zero point. The zero point may be located at the electrode tip , the top face , the reduction bottom edge , the electrode holder or on the electrode holder coupling at the machine axis .

Direction to rotate the derived electrode: For the purpose of derivation, electrodes are rotated by 180° for CAM programming. Specify whether they are to be rotated around the X axis or Y axis of the reference system. The option must be selected before creating an electrode.

When selecting the X Z or Y Z option, the electrode is additionally rotated 90° about the Z axis. In this way, a transponder chip (code or ID of an electrode in the electrode holder) can be rotated to automate tipping for milling, so that it is always correctly positioned towards the scanner to determine the RFID code.

Prefix for current document name for derived electrode: Identify file names for documents containing derived electrodes by means of a prefix. Only if no folder is selected in the Folders → Electrode project menu item.

Prefix text for derived electrode: Enter a prefix for the file names for documents with the derived electrodes.

Prefix for current document name for report: Prefix file names for electrode and raw material reports.

Prefix text for report: Enter a prefix for the file names for electrode and raw material reports.

EDM

Machine head for collision check: Select the machine.

A dynamically adapted drop-down menu of eroding machines is displayed that are defined in the file selected in the XML file for machine heads option. If a name is selected, the head is checked for collisions and information is displayed, otherwise a warning is output indicating that the machine head is not checked.

For electrodes without an electrode holder, a machine head is not displayed and not checked.

VDI surface tolerance: Select a value for the VDI surface tolerance.

Orbit type: Select the shape of the electrode orbit type (e.g., sink, sphere, half sphere, square, widen, linear, sink and widen, sink and sphere, sink and square, injection, ISOG) when eroding. A dynamically customized drop-down menu of the orbit types defined in the file selected in the XML file for orbit types option is displayed. Also see User-defined orbit type.

Clearance plane reference system: Select the reference system for the clearance plane.

Clearance plane offset: Enter a distance in Z to be maintained above the workpiece for electrode processes.

Dielectric: Specify whether a dielectric is used.

Dielectric height: Enter the height for the dielectric fluid level.

Dielectric reference: Specify the reference point for the dielectric height.

EDM C angle interval type: Select the rotation angle interval for the C axis (0° - 360° or -180° - 180°).

EDM priority: Select the EDM priority (low wear, normal, high removal, view structure).

Maximum magazine positions: Enter the maximum number of electrode slots in the electrode changer.

Wear division: Enables a function on Agie erosion machines that reduces electrode wear.

Folders

Note

The supplied files are examples. Use these examples to add your own data. For more information, see the Company-wide default settings section.

Holders and XML files: Enter the folder path for the electrode color palette and the raw material, electrode holder and machine head file.

%HC_PRODUCT_FOLDER% returns the location of the software product that has been called as a root folder.

Electrode project: Select a folder for the milling project to create derived electrodes and for milling programming.

If no electrode project folder has been selected, the folder of the opened *.hmc document will be used as the folder for the electrode project data.

If an electrode project folder has been entered, a subfolder with the name of the electrode is created for each derived electrode.

The derived electrode is stored in this folder. A check is carried out to make sure that the folder and electrode names match, and that no folder is overwritten or created multiple times. All data for CAM programming is stored in the folder of the relevant electrode and only appears there once. Reports are saved in the root folder of the milling project. This means that the folder can be used as a root for backup copies.

Export reports: Select a folder to permanently store electrode reports that are generated with the Export reports command.

XML file for raw material and holder: Select the file prepared in the ‘Blockdataeditor’ program. This file contains all available raw material and electrode holders.

XML file for machine heads: Select an electrode_machine.xml file that has been prepared with the information on the machine heads for the collision check.

XML file for electrode color import: Select the *.hcmaterials.xml color library with the electrode color palette.

XML file for orbit types: Select an Electrode_orbit.xml file that contains the relevant orbit type data. Also see User-defined orbit type.

XML file for materials: Select a default.densities.xml file that contains the relevant information on materials. Also see User-defined material list.

XML file for generator program: Select an electrode_generator_programs.xml file with generator programs for performance parameters for eroding. Also see Preparing generator programs.

Colors

Select the colors for the Contact face , the Erosion faces , the Extensions of the electrode shaft, the Reduction top face , the Reduction , the Block and the Raw material as well as for the Electrode holder .

Select the color for Reduction bottom face and First solid bottom face . These faces can be excluded from the color selection for CAM programming by assigning separate colors for the two faces. When a document with previous color definitions is used, both colors are automatically added.

Select the colors for the Joint and the Joint top face of a side electrode. Assigning a separate color for the Joint top face makes it possible to exclude these faces from the color selection for CAM programming. When using a document with previous color definitions, both colors are automatically added.

Exploded view using the default color palette for electrodes

You have the option of coloring particular faces of the electrode using User defined 1 through to User defined 5, for example, for exposed areas, pockets or supports.

Load predefined colors using the Import colors command.
