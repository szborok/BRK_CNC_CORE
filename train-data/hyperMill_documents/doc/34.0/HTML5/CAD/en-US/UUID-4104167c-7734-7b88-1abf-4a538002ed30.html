Own file formats

Interface formats of OPEN MIND software products.

*.hmc

Document format

*.hmup

Format for saving markups.

*.hmct

Format for saving a geometry as a template for a new document

*.hmcgeom

Native format containing geometric and topological data in binary, uncompressed form including information on solids and meshes.

In addition, basic graphic properties (color, line type and thickness, layer name) are also saved.

*.hcplain

Uncompressed document format

*.bnd

BND is a hyperMILL® format. hyperMILL® boundaries can be imported.

The entities are inserted in the model tree. A group can be optionally created around the imported data. The data is given its own layer with the name of the file or with a specially specified layer name.

Table 25. Options for opening a *.bnd file

Property

Description

General

Entity precision: Enter a value that is used for the same-named Entity precision property of the CAD entities.

Layer: Specify an own layer name.

Color: Overwrite the current color with RGB value during import.

Group entities: Create a group around the imported data.

Unit of measure: Read in data in the Millimeter unit of measure or Inch unit of measure. Select the unit of measure.

Log file

Log file mode:

  • Same as input file: Save the log file with the file name of the model in the same folder as the model file.

  • Select path: Select another folder for the log file.

  • Select file: Specify a different file name for the log file.

  • None: Do not generate a log file.

Log file path: Specify a path for the log file.

Log file name: Specify a file name for the log file.

Show log file after import: If a log file is generated, automatically display the file after import.



The settings are stored in the intfmtoptbndvalues.xml file.

*.omx

OMX is a hyperMILL® format for geometry data. Geometry data such as boundaries or drill positions can be transferred from one job list to another.

The entities are inserted in the model tree. A group can be optionally created around the imported data. The data is given its own layer with the name of the file or with a specially specified layer name.
