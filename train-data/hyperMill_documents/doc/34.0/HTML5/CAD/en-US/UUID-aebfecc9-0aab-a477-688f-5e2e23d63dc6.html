16.23. Preparing the eroding machine

In order to include a die-sinking EDM machine head in collision avoidance when generating an electrode, a sufficiently precise geometric description must be stored and managed. Record all machines that are to be available for selection in the Machine head for collision check option in a electrode_machine.xml file. This file contains references to *.hmc documents containing geometries of the machine heads.

For a motion simulation that includes an eroding path collision check with a complete eroding machine model generated using the Modify eroding path command, which is located in the SIMULATION Center, include further references to *.vmm files in the electrode_machine.xml file.

Under .../hyperCAD-S/files/electrode/machine, save the *.hmc documents with the geometries:

Split the geometry into layers in accordance with the command. Position the workplane so that the electrode holder fits the adapter and the correct length of the combination is obtained.

  • On one layer with the name from the XML entity <CAxisLayer>: All solids that adequately describe the clamping system that holds the electrode holder. The clamping system is rotated like the electrode holder according to the C angle.

  • All remaining layers: All solids that describe the machine axis with sufficient accuracy. The alignment refers to the reference system and is static.

Under .../hyperCAD-S/files/electrode/machine save the *.vmm file with the complete eroding machine model. To obtain such machine models, contact an OPEN MIND sales partner.

The files located in the .../hyperCAD-S/files/electrode/simulation folder do not have to be adapted and must not be changed. These files are required for the respective concrete simulation and are processed automatically.

Under .../hyperCAD-S/files/electrode, save the electrode_machine.xml file with the following structure.

<?xml version="1.0"?>
<EDMMachines>
    <EDMMachine name="Machine name">
        <filename>Machine\Machinename.hmc</filename>
	<CAxisLayer>Clamping device</CAxisLayer>
        <SimulationModelFile>Machine\Machinename.vmm
        </SimulationModelFile>
    </EDMMachine>
</EDMMachines>

Enter the name of the eroding machine under <EDMMachine> in the name attribute. This name is displayed for selection in the dialog. Under <filename>, enter the folder and the *.hmc document that contains the geometry data for the machine head. Under <CAxisLayer>, enter the layer name containing all solids for the electrode holder’s clamping system.

Under <SimulationModelFile>, enter the *.vmm file with the EDM machine’s complete machine model.

Example 25. XML file with two die sinking EDM machines and the same clamping system
<?xml version="1.0"?>
<EDMMachines>
    <EDMMachine name="S GE 400">
	<filename>Machine\SGE400.hmc</filename>
	<CAxisLayer>S3R</CAxisLayer>
	<SimulationModelFile></SimulationModelFile>
    </EDMMachine>
    <EDMMachine name="CM 323 B">
	<filename>Machine\CM323B.hmc</filename>
	<CAxisLayer>S3R</CAxisLayer>
	<SimulationModelFile></SimulationModelFile>
    </EDMMachine>
</EDMMachines>


Example 26. XML file with die-sinking EDM machine for simulation
<?xml version="1.0"?>
<EDMMachines>
    <EDMMachine name="ERODICON_EDM01">
	<filename>Machine\Erodicon_01.hmc</filename>
	<CAxisLayer>Layer_0001</CAxisLayer>
	<SimulationModelFile>Machine\Erodicon_01.vmm</SimulationModelFile>
    </EDMMachine>	
</EDMMachines>


Important

The machine models supplied are examples and should not be used for collision checking in production.

Theelectrode_machine.xml file is required even without a machine. The content can be reduced to the following:

<?xml version="1.0"?>
<EDMMachines>
</EDMMachines>

The electrode reports show whether the machine head has been checked for collisions.

Important

If the machine is changed within the Modify EDM parameters command, the machine head is no longer checked for collisions.

The machine head is set to not checked if the machine name is changed.

The machine head is displayed in the preview, but is not added to the model.
