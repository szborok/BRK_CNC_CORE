16.18. Electrode holder and raw material editor

Manage electrode holders and raw materials.

Electrodes → Electrode holder and raw material editor

Electrode

The electrode holder and raw material editor is a separate software program named blockdataeditor.exe for managing the combinations of electrode holders and raw material that exist in production.

File

Create, open and save an electrode holder / raw material library as an *.xml file.

Select the last used documents directly for opening.

Edit

Enter a new electrode holder using Add electrode holder. Select the prepared *.hmc electrode holder file. Remove an electrode holder from the library using Remove raw material and electrode holder.

Enter the dimensions for new raw material using Add raw material. Delete raw material using Remove raw material.

Double-click with the left mouse button in the field of the appropriate column to fill out or change values. Or call the command Edit on the context menu.

Important

Electrode holders are used in the collision model for the collision check. The electrode holders are surrounded by a bounding box for each solid, which are used as the basis for collision checking.

Configure the model geometry for the electrode holder:

  • Optimize the level of model detail for precision and calculation time. To do so, prepare the electrode holder geometry.

  • Place the world workplane centered on the support surface of the raw material in the electrode holder .

  • Orient the electrode holder so that the electrode mount (e.g., holes or slot) lies in direction of the X or Y axis (principal orientation).

Preparing electrode holder geometry

Tips to prepare electrode holder geometry, as they can be imported as 3D geometry from manufacturers.

Data optimization is necessary. Every solid in an electrode holder is used for the collision check. If the data is not prepared, more time is needed to check and insert. This can be avoided:

  1. Reduce the geometry to the key information about the electrode holder needed for the collision check. Delete all entities except the holder faces ('shell').

    Edit → Delete

  2. Check for redundant entities. Delete double faces.

    Analysis → Check double entities

  3. Convert NURBS entities to analytical geometry.

    Modify → Convert to analytical

  4. Merge 'fragmented' faces into an entity Simplify the faces.

    Modify → Simplify

  5. Create solids. Create as few solids as possible. Only the characteristic base shapes are captured as a solid. A maximum of 3 solids is recommended.

    Shapes → Create solid

Command line parameters for the ‘Blockdataeditor’ program

-i Enable large preview images.

-o [PATH/NAME.xml] Open the specified *.xml file. Can also be specified at the end of the parameters or as a single parameter without -o.

-d [PATH] Set the start folder.

-l [language code] Use the specified language instead of the locale.
