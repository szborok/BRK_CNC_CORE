Data backup

Experience shows that data losses will always occur. It is only a matter of time before such a loss takes place. There are many different reasons for this, for example, a power failure, the theft of a device, a hardware defect or a user error.

Data belongs to the operating capital of a company. Missing or incomplete data can not only lead to material damage and damage to the image of a company, great effort is also required to recover the data. Rules on liability, contractual penalties and proper business operation must be observed.

Users of OPEN MIND software are fully responsible for backing up data through the provision of systems and methods to avoid the loss of data.

The following four points, for example, can be used to check the data backup status:

  • Are backup copies generated? Where are they stored?

  • After which time interval does the data change to a relevant extent? Is the storage frequency suitable for this?

  • How quickly does the backed-up data need to be available again after a data loss?

  • Is the recovery of data tested successfully on a regular basis?

Options for generating backup copies can be selected under File → Options → Options / properties. Further information is provided in Application.

Basic information can be found on Wikipedia, for example, https://en.wikipedia.org/wiki/Backup.
